# Глоссарий — hermes-lifemodel

Термины проекта одним местом. Для «технаря»: коротко и по делу, с указанием, откуда термин (биология/психология/наша инженерия).

## Слои и поток

- **CoreLoop (сердцебиение / heartbeat).** Главный цикл-планировщик. Владеет тиком, распределяет «процессорное время» компонентам, тратит энергию, ловит сбои. Живёт в главном потоке, неубиваем. Аналог: CPU + шина + предохранители. Тик у нас ~60с (пол среды Hermes).
- **Тик (tick).** Один удар сердца. Каждый тик прогоняются дешёвые слои; дорогие — только если проходит гейт по энергии/порогу.
- **Канал (channel).** «Орган чувств» — вход из мира (Telegram, CLI). Даёт Hermes.
- **Сигнал (signal).** «Нервный импульс» — афферентная единица потока. **Эфемерен: живёт ≤ один `ExecutionFrame`, in-memory, не персистится** (шина пустая после рестарта — core-rethink 2026-07-11). Только чтение; мутации — через **интенты**. Закрытая таксономия: `ObservationSignal` (контакт/доставка/время) · `DriveSignal` (`u`) · `CognitiveOutcome` (`sent/silent/failed/stale`) · `ProposalSignal` (мысль предлагает контакт, позже).
- **Слой (layer).** «Отдел мозга». Три штуки, снизу вверх — дороже и умнее:
  - **AUTONOMIC (автономный, ствол мозга).** Нейроны, 0 LLM. Мониторят состояние, эмитят сигналы. Всегда работают, стоят 0 энергии.
  - **AGGREGATION (агрегация, таламус).** 0 LLM. Копит сигналы, считает **важность (salience)**, ведёт жизненный цикл желания, решает «будить ли когницию». Тоже 0 энергии.
  - **COGNITION (когниция, сознание).** LLM: быстрая модель → умная по неуверенности. Единственный слой, который тратит энергию.
- **Motor / моторика.** Выполнение действий в мире (sandbox Hermes). Не слой мозга, отдельный сервис. Пока не строим.

### Пересборка сенсорного ядра (core-rethink 2026-07-11)

Чёткое разделение ролей, которые раньше были смешаны под словом «сигнал/нейрон». Полный дизайн — [`superpowers/specs/2026-07-11-sensory-core-rethink-design.md`](superpowers/specs/2026-07-11-sensory-core-rethink-design.md).

- **`ExecutionFrame` (кадр исполнения).** Единица обработки — **не только heartbeat**: frame запускает heartbeat ∨ входящее событие ∨ **завершение async-когниции** ∨ admin-команда. Каждый frame: снапшот state → `SignalFrame` (эфемерная шина) → свёртка → интенты → **атомарный коммит в конце frame** → сигналы умерли. Все frame-ы **сериализованы одним state-actor**. Сигнал живёт ≤ один frame.
- **Организм/Канал-адаптер (`OrganAdapter`).** Интеграция Hermes (Telegram…). **Двунаправлен:** афферентная сторона = сенсоры (вход), эфферентная = мотор/egress (выход). *Двунаправлен канал, НЕ сенсор.*
- **Сенсор (`Sensor`).** Афферентный трансдьюсер: читает вход канала → сигнал. **Band-pass фильтр на самом сенсоре** (control-команды не проходят — как ухо не слышит ультразвук). Тупой. ⚠️ **Сенсор ≠ нейрон ≠ драйв.**
- **Драйв (`Drive`).** Гомеостатический/аффективный интегратор (Панксепп PANIC/GRIEF): копит `u` во времени (растёт в **тишине** = отсутствии контакта, `satiate` на контакт). **НЕ сенсор** — мотивационный центр **с состоянием** (`u` durable в `AgentState`). Аналог: голод, а не «сенсор еды».
- **`AgentState` (StatePort) vs `UserModel` (MemoryPort).** «Я» (авторитетно, всегда свежо, без TTL: `u`/energy/mood + контакт-бухгалтерия) vs «Другой» (**наш вывод/кэш**, пер-поле `{value, inferred_at, ttl}`; протухло → «неизвестно»). Бывший `Relationship` → `UserModel`. Сырьё для вывода — провайдер (hindsight/Honcho).
- **`contact_observed` / `proactive_outcome`.** Доменные факты (бывшие `exchange`/`verdict` — переименованы, не удалены): реальный контакт наблюдён; исход собственного хода (`sent/silent/failed/stale`).

## Нейроны и агрегация

- **Нейрон (neuron).** Тупой сенсор: измеряет один аспект состояния и эмитит **сырьё** `{значение, дельта, длительность-над-порогом}`. **Логики оценки важности в нём нет** — это работа агрегации.
- **Refractory-период.** «Не долблю постоянно» — минимальный интервал между эмиссиями нейрона. Считается **по часам**, не по тикам.
- **Habituation (привыкание).** Снижение важности сигнала, который долго держится на одном уровне. «К хроническому привыкаешь».
- **Weber–Fechner (детекция изменений).** Значимо **относительное** изменение: +10% заметны и при 0.1, и при 0.9. Основа расчёта salience.
- **Salience (важность / салиентность).** Насколько сигнал важен **сейчас** = амплитуда + скорость изменения − привыкание. Считает **агрегация** (у неё контекст всех сигналов). Общая «валюта сравнения» разнотипных сигналов.
- **Желание (desire).** Объект с жизненным циклом, которым владеет агрегация: **рождение** (первый переход порога) → **дедуп/ack** (пока живо, повторные сигналы поглощаются) → **разрешение** (исполнить/отложить/отвергнуть — по `proactive_outcome` завершённого хода) → **release** отложенного (присутствие / доступность / депривация). *(«вердикт» как термин выведен — см. `proactive_outcome`, core-rethink 2026-07-11.)*
- **ack / дедуп.** Свойство живого желания: повторные импульсы не создают дубль и не будят когницию заново.
- **Порог (threshold).** Точка на шкале, за которой агрегация будит когницию.
- **Backpressure / сброс нагрузки (load-shedding).** Защита агрегации от потока «орущих» сенсоров: слой ограничен по работе на тик (`MAX_INTAKE`), схлопывает однотипные (коалесинг), а при перегрузке отбрасывает наименее важные сигналы (по salience) — и **никогда не отказывает**. Биоаналог: сверх-боль → организм гасит сознание, а не ломается.
- **Коалесинг (coalescing).** Схлопывание однотипных сигналов в один (свежайший/важнейший + счётчик слитых), чтобы уменьшить объём до обработки.
- **Ватерлиния / гистерезис (watermark / hysteresis).** Режим сброса включается выше `high_watermark`, держится **до** падения ниже `low_watermark` — «до возвращения в норму», без дрожания на границе.

## Решение и речь

- **Intents (интенты).** Слои **не меняют состояние сами** — возвращают намерения (`UPDATE_STATE`, `SEND_MESSAGE`, `WAKE_COGNITION`…), а применяет их CoreLoop атомарно. Даёт тестируемость и откат.
- **Wake-packet.** Пакет для разбуженной когниции: кто/почему разбудил (сигнал, значение, порог), состояние, уместность. Сырые аффекты в LLM не уходят — переводятся в человеческую формулировку.
- **Act-gate.** Вердикт разбуженной когниции: **исполнить / отложить-и-держать / отвергнуть** (`[SILENT]`/`NO_REPLY`). Уместность момента решает когниция (у неё знание о человеке), а не дешёвый гейт.
- **Пробуждение ≠ отправка.** Переход порога лишь будит мышление; отправит ли сообщение — решает act-gate.

## Драйвы, личность, эмоции

- **Драйв (drive).** Внутренняя нужда, которая копится и толкает к действию. Наш v1-драйв — **контакт**.
- **Гомеостаз / homeostatic-RL.** Терморегулятор: есть **целевой уровень (set-point)**; отклонение = «ошибка» (нужда); действие, уменьшающее ошибку, = «награда». `ошибка = set-point − текущее`.
- **Панксепп (primary affective systems).** Нейронаучная модель: в мозге несколько встроенных «мотор-демонов»: SEEKING (искать), CARE (заботиться), PLAY (играть), FEAR (бояться), RAGE (злиться на преграду), **PANIC/GRIEF (тоска разлуки)**. Наш контакт-драйв = PANIC/GRIEF: одиночество копится в изоляции, гасится реальным контактом. **Движок драйвов.**
- **SDT (Self-Determination Theory).** Психология: 3 нужды — автономия, компетентность, **связь**. У нас — только **человеко-читаемый язык** и фильтр перед промптом, НЕ движок.
- **Личность (self-model).** Компонент: держит темперамент (конфиг), быстрые состояния (энергия, настроение…), медленно-выученное (set-point'ы).
- **Темперамент.** Статичный конфиг «кто это по натуре»: веса драйвов, ёмкость энергии, чувствительности.
- **Валентность (valence).** Знак состояния: хорошо↔плохо (`−1..1`). Драйвы — униполярны (`0..1`, отрицательного одиночества не бывает); настроение/receptivity — биполярны.
- **Настроение (mood) — циркумплекс Рассела.** 2 оси вместо ярлыков: **валентность** (хорошо↔плохо) × **арузал** (спокоен↔взвинчен). Любое настроение = точка.
- **Арузал (arousal).** Уровень активации/возбуждения `0..1`.
- **Receptivity (расположенность).** Насколько сейчас уместен контакт (со стороны пользователя — в UserModel; как состояние существа — валентное).
- **ActionPending (опережающее торможение).** Пока ждём ответа на свой выход — рост драйва **подавляется**, чтобы к моменту ответа не быть на пике.

## Энергия и физиология

- **Энергия (energy, E).** «Аккумулятор». Тратится на **дорогие** действия (когниция/моторика), восстанавливается в покое. Дешёвые слои её **не тратят** (ствол мозга не исчерпывается).
- **Двухпроцессная модель сна (Борбели).** Почему клонит в сон:
  - **S — долг усталости:** растёт от бодрствования/работы, гасится покоем.
  - **C — циркадный ритм:** ~24ч волна бодрости, **функция абсолютного времени** (не по тикам).
- **Йеркс–Додсон.** Производительность лучше при **среднем** арузале (перевёрнутая-U): и мало, и слишком много — плохо.
- **Ego-depletion («сила воли как жижа»).** Развенчанная теория. **Не моделируем.** Усталость — это S+C, а не «кончилась воля».
- **Резерв → сверка.** Перед дорогим ходом резервируем **ожидаемую** цену (гейт «хватает ли»), после — сверяем **фактическую** (цену LLM знаем только после). Перерасход может увести E в минус → форс-восстановление.
- **Прогрессивная деградация.** При нехватке энергии редеет только когниция (дорого) — сама, из экономики бюджета, без явного `if energy<X`. Рефлексы живут всегда.

## Пользователь и наблюдаемость

- **UserModel (модель пользователя).** Байесовская оценка **с неопределённостью** твоего состояния: доступность, receptivity, ритм (4 бакета: утро/день/вечер/ночь). Обновляется от наблюдаемого (скорость/качество ответа, игнор, инициатива). Ума не читает.
- **Качество контакта.** Не только скорость: быстрый «ок» ≠ успех. Метрика вовлечённости (напр. токены_юзера/токены_агента) → насыщение + UserModel.
- **Уместность (appropriateness).** `salience желания × доступность × receptivity × обратная-цена-прерывания`. Вход для act-gate.
- **Наблюдаемость (observability).** Структурные логи + трейсы (`trace_id`/`tick`/`correlation_id`). Каждый проактивный выход отвечает: какой сигнал/порог/желание/гейт/что бы заблокировало. Инструмент — `/lifemodel debug`.

## Инженерия и безопасность

- **Composition root.** Единая точка сборки (DI): руками связывает компоненты и регистрирует их. Ядро **не импортирует типы плагинов**.
- **Реестр (registry).** Куда компоненты (нейроны/слои) **самрегистрируются** через DI-колбэки; можно **вкл/выкл**.
- **Манифест компонента.** Внутренний дескриптор `{id, type, enabled, version, config}`.
- **Blackboard.** Компоненты не общаются напрямую (нет циклов) — только через шину сигналов («общая доска»).
- **Checkpoint (снапшот сознания).** Атомарный снимок состояния всех компонентов одним blob-ом (tmp+rename).
- **Circuit-breaker.** Сбоящий компонент отключается, существо «живёт без органа» и объясняет деградацию.
- **Глобальный предохранитель (safety backstop).** Жёсткий лимит реальных отправок (N/сутки + мин.интервал), **fail-closed**, поверх всех слоёв. **Не механизм сдержанности** (та эмёрджентна), а защита пользователя от сбойной/галлюцинирующей LLM.
- **fail-open / fail-closed.** Сенсоры/драйвы при сбое — **пропускаем** (живём дальше); безопасность/enforcement при сбое — **не выполняем** действие.
- **Cold-start / bootstrap.** Первые дни существо не знает ритмов/set-point'ов → стартовый консервативный темперамент, чтобы не молчать и не спамить.
- **Dithering (микрошум).** Небольшой шум к порогам/таймингам, чтобы поведение не было «по будильнику» алгоритмичным.
- **«Барабан» (drum).** Старый баг: слепой таймер слал «нечего сказать» ровным боем. Убран **как класс** правильной архитектурой (тишина по умолчанию), а не подавителями.

## Среда

- **Hermes.** Агент-фреймворк-«тело»: даёт транспорт, планировщик (cron), память (hindsight), sandbox. Мы — «нервная динамика» поверх.
- **Порт / адаптер (ports & adapters).** Граница: ядро зависит от портов (`DeliveryPort`, `MemoryPort`, `LlmPort`, `StatePort`, `ClockPort`), Hermes — за адаптерами. Ядро host/model-agnostic.
