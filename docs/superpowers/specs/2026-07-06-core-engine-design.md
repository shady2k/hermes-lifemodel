# Дизайн ядра — слоёный движок «цифрового человека» (пересборка)

**Родословная:** конкретизирует `docs/hla.md` §1–2 до реализуемого ядра; исходное видение — оригинальный `lifemodel` (github.com/shady2k/lifemodel).

## 1. Контекст и цель

`lifemodel` — проактивный агент как **«цифровой человек»**: тянется к человеку сам, когда внутри копится нужда, а не отвечает по запросу. Ведущая метафора — **тело как инженерное ограничение**: биология диктует энергосбережение и слоёную обработку.

**Проблема.** Живая реализация выродилась в **монолит одного драйва «контакт»** (`core/decision.py`), воткнутый в тик и обошедший слоёную архитектуру (пустой список нейронов, `SilentAggregator`). Это дрейф от HLA.

**Цель.** Пересобрать **настоящее слоёное ядро** (AUTONOMIC → AGGREGATION → COGNITION с живой шиной, планировщиком и энергией), переселив контакт-драйв **первым нейроном внутрь слоёв**. Ядро — чистое, DI, компоненты-плагины, готово к росту (мысли, обязательства, настроение, моторика, сон) без переписывания.

## 2. Принципы (законы, не пожелания)

1. **Логика — в слоях, не в нейроне.** Нейрон — тупой сенсор: измеряет и эмитит. Решения — у слоёв выше.
2. **Ниже = глупее и дешевле.** Нижний слой не может быть умнее верхнего. В этом смысл слоёв: экономить энергию, включая дорогое только по порогу.
3. **Слои не мутируют состояние.** Они возвращают **Intents**; применяет их CoreLoop (§6).
4. **Сдержанность эмёрджентна.** Тишина — состояние по умолчанию; существо говорит, только когда реальное давление перешло порог И когниция сочла, что есть повод. Инварианты безопасности хранятся **как тесты**, не как «фича подавления» (§12). Поверх всего — один жёсткий предохранитель (§14).
5. **Без костылей и быстрых побед.** Делаем правильно; временные заглушки — только явно помеченные и с задачей на замену.
6. **CoreLoop неубиваем.** Сердце в главном потоке никогда не падает и не тормозит; компоненты — асинхронные и изолированные (§7).
7. **DI + гексагон.** Ядро зависит от портов; Hermes — за адаптерами; ядро **не импортирует типы плагинов**.
8. **Fail-open для сенсоров/драйвов, fail-closed для безопасности** (наследие HLA §7).

## 3. Слоёный конвейер

```
Каналы(чувства) → Сигналы → AUTONOMIC(нейроны, 0 LLM) → AGGREGATION(0 LLM) → COGNITION(LLM) → Intents → CoreLoop применяет
```
- **AUTONOMIC** — нейроны: мониторят состояние, эмитят сигнал `{value, delta, duration_over_threshold}`. Ствол мозга. **Стоят 0 энергии, работают каждый тик.**
- **AGGREGATION** — шина сигналов, считает **salience** (§5), habituation, владеет **жизненным циклом желания** (§12), решает порог пробуждения. Таламус. **Тоже 0 энергии, каждый тик.**
- **COGNITION** — LLM (fast → smart по неуверенности); **act-gate** = вердикт «исполнить / отложить / отвергнуть». Сознание. **Единственный потребитель энергии среди слоёв** (§8).

> **Поправка внешнего ревью (ловушка комы):** энергия гасит **только** дорогое (когниция/моторика). Дешёвые слои **бесплатны и всегда on** — иначе при нуле заряда существо не заметит, что выспалось, и не обработает сигнал пробуждения.

## 4. Сигналы и шкала

Сигнал — единый язык («сигналы, не события»).

- **Шкала нормализованная и типизированная:** **униполярно `0..1`** для драйвов/дефицитов (отрицательного одиночества не бывает); **биполярно `−1..1`** для валентных состояний (настроение/receptivity). Валентность — first-class.
- **Нейрон эмитит только сырьё:** `{value, delta, duration_over_threshold, confidence}`. Никакой оценки важности.
- **Реестр таксономии сигналов:** каждый вид объявляет `{schema, polarity, ttl, privacy}`. Нужен до сторонних компонентов, чтобы ядро знало, как обрабатывать чужой сигнал.
- **Дедуп по origin-id** с TTL (сообщение не считается дважды — как сигнал входящего хода и как сигнал следующего тика).
- Форма сигнала: `id, kind, source(category.specific), polarity, value, metrics{delta, rate_of_change, confidence}, priority, ts, ttl, correlation_id, parent_id`.

## 5. Salience — считает АГРЕГАЦИЯ, не нейрон

Важность требует контекста всех сигналов во времени → это работа агрегации (нейрон не может быть умнее слоя выше).

**Salience ≠ только амплитуда** (уточнение внешнего ревью, совпадает с Weber-Fechner из оригинала):
```
baseline = EMA(value)                       # скользящая «норма»
salience = w1·value + w2·|rate_of_change| − w3·habituation
```
Медленно доросшее до 0.9 → привыкли (низкая salience); скачок 0.2→0.8 за час → важно. Salience — общая «валюта сравнения» разнотипных сигналов; сам `value` разнотипных сигналов складывать нельзя.

## 6. Intents (слои не мутируют state)

Каждый слой возвращает список интентов; **CoreLoop применяет их атомарно** в конце тика. Даёт тестируемость, трассируемость, откат.

Набор v1: `UPDATE_STATE`, `EMIT_SIGNAL`, `ACK_DESIRE`, `DEFER_DESIRE`, `WAKE_COGNITION`, `SEND_MESSAGE`, `SPEND_ENERGY`, `UPDATE_USER_MODEL`. Расширяется по мере фаз.

## 7. CoreLoop — сердце и планировщик (как CPU)

- **Главный поток, неубиваемый.** Никогда не падает/не тормозит. Оборачивает **каждый** вызов компонента: исключение — компонент пропущен в этот тик; повторный сбой — circuit-breaker (существо «живёт без органа», объясняет деградацию).
- **Компоненты — асинхронные, в отдельном исполнителе/потоке.** Даже теоретически не могут уронить ядро. (В среде Hermes: тик крутится на защищённом контуре, тяжёлые вызовы вынесены из главного потока.)
- **Планировщик:** дешёвые слои — каждый тик; дорогие (когниция/моторика) — **по бюджету энергии** (§8).
- **Прерываемость (уточнение внешнего ревью):** проактивная когниция несёт `cancellation_token` и `timeout`. Реальное сообщение пользователя = **рефлекторная дуга**: ядро вправе прервать/перепланировать дорогой ход. (Частично это уже делает FIFO входящих в Hermes.)
- **Ловушка «вечной мысли»:** «подумать-но-промолчать» тратит энергию и не снижает salience. Защита — жизненный цикл желания (§12): отложенное желание **паркуется** (не будит когницию повторно до release-условия), а не переоценивается каждый тик.

## 8. Энергия — физиология (по науке, не «сила воли»)

Модель: **двухпроцессная (Борбели)** + **арузал (Йеркс-Додсон)**. Ego-depletion развенчана — не моделируем.

- **S — долг усталости** `[0..1]`: растёт от бодрствования и вычислений, гасится покоем.
- **C — циркадный ритм:** **функция абсолютного времени** (Unix ts), не накопление по тикам → иммунитет к пропущенным тикам. `C(t)=0.5+0.5·cos(2π·(t−φ)/24h)`.
- **A — арузал** `[0..1]`: перевёрнутая-U (и мало, и слишком много — плохо).
- **E — энергия** `[−…1]`: тратится на **дорогие** действия, регенерирует в покое (сильнее при ночном C).
- **Только дорогое стоит энергии.** Autonomic/aggregation = **0** в энергоединицах (ствол мозга не исчерпывается). Энергия = «ОЗУ когниции».
- **Резерв → сверка:** перед дорогим ходом CoreLoop **резервирует ожидаемую цену** (гейт «хватает ли»), после — **сверяет по факту** (цену LLM знаем только после). Перерасход **допускает уход E в минус** → форс-восстановление дольше обычного («отрубился от истощения»).
- **Прогрессивное «отключение» — эмёрджентно:** запускаем дорогое, только если `ценность/цена` высока и `цена ≤ бюджет`. Сел заряд → редеет только когниция; рефлексы живут.
- **Прайс-лист** (v1 — фикс, позже — обучаемый): `fast_llm ≈ 0.02`, `smart_llm ≈ 0.08`, `send ≈ 0.03`, `tool ≈ 0.04+`.

## 9. Компонент «Личность» (модель себя)

Держит три вида характеристик:
- **Темперамент (конфиг, статичен):** веса драйвов, ёмкость энергии, скорость восстановления, чувствительности. + **bootstrap-профиль** для cold-start (первые дни: консервативный prior, чтобы не молчать и не спамить).
- **Быстрые состояния (каждый тик):** энергия, долг усталости, арузал, настроение, дефицит контакта.
- **Медленно-выученное (позже):** set-point'ы.

**Движок драйвов — первичные аффективные системы Панксеппа** (биологичны, валентны: SEEKING/CARE/PLAY/FEAR/RAGE/PANIC-GRIEF). Контакт-драйв = **гомеостат сепарационного дистресса (PANIC/GRIEF)**: одиночество копится в изоляции, гасится реальным контактом. **SDT** (автономия/компетентность/связь) — только человеко-читаемый **язык и фильтр-интерпретатор** (§13), не движок.

**Настроение — циркумплекс Рассела:** валентность `−1..1` × арузал `0..1`; заземлено в наблюдаемых сигналах; смещает приоритет/тон; **фактов не выдумывает**. (Компонент-производитель настроения в v1 не строим — только шов; см. §19.)

**ActionPending (уточнение внешнего ревью):** пока ждём ответа на свой выход — **рост драйва подавляется** (feedforward inhibition), иначе к моменту ответа существо на пике одиночества.

## 10. Компонент «UserModel» (модель пользователя)

Байесовская **оценка с неопределённостью** из **наблюдаемого** (не читаем ум):
- `availability [0..1]`, `receptivity [−1..1]`, `rhythm` — **4 бакета** (утро/день/вечер/ночь) в v1, не матрица 24×7.
- **Качество контакта, не только скорость** (уточнение внешнего ревью): быстрый «ок/ясно» ≠ успех. Метрика вовлечённости (напр. `tokens_user/tokens_agent`) считается в агрегации → кормит насыщение и UserModel. Это наш `q_event`-классификатор.
- Апдейт: `availability` затухает к приору по времени суток и подтягивается свидетельствами; `receptivity` растёт на инициативу/тёплый ответ, падает на игнор/отказ.
- Выдаёт **уместность** = `salience_желания × availability × norm(receptivity) × обратная_цена_прерывания` — вход для act-gate.

## 11. Нейроны (контракт + ContactNeuron)

- **Контракт:** `check(state, now, correlation_id) -> Signal | None`. Тупой сенсор.
- **BaseNeuron:** refractory-период (**по часам**, не по тикам), детектор изменений (Weber-Fechner), `last_value` для наблюдаемости.
- **ContactNeuron (v1, единственный):** переселяет **сертифицированную математику драйва** (rise-в-тишине / satiate-на-контакте) внутрь нейрона; эмитит `contact_pressure` сигнал `{value, delta, duration_over_threshold}`. Никакого backoff/дедупа внутри нейрона — это агрегация (§12).

## 12. Агрегация (жизненный цикл желания + инварианты)

Агрегация владеет **желанием как объектом с жизненным циклом** (HLA §2.1):
1. **Рождение:** первый переход порога (+структурные проверки) → создаёт **одно** желание → `WAKE_COGNITION`.
2. **Дедуп/`ack`:** пока желание живо (в т.ч. «отложено») — повторные сигналы поглощаются, когниция повторно не будится.
3. **Вердикт когниции:** исполнить → контакт случился, давление спадёт само; отложить → держим, давление не сбрасываем, ждём release; отвергнуть → убрать + короткий backoff.
4. **Release отложенного:** присутствие пользователя ∨ высокая доступность ∨ депривация-эскалация по времени.

**Инварианты сохраняем как ТЕСТЫ (не как «фичу подавления»):** одно желание на переход; дедуп живого; «отложить» не сбрасывает давление; «отвергнуть» → backoff; насыщает только реальный контакт; свой импульс ≠ контакт; **пробуждение ≠ отправка**. Сертифицированные сценарные тесты старого драйва **переносятся на границу слоёв**.

## 13. Когниция (wake-packet, act-gate, prompt-safety)

- **Wake-packet** = soul-packet + повод пробуждения (какой сигнал/значение/порог, состояние, последний контакт, уместность из UserModel).
- **Act-gate** = вердикт: **исполнить / отложить-и-держать / отвергнуть** (`[SILENT]`/`NO_REPLY`). Уместность момента решает **разбуженная когниция** (у неё знание о человеке), а не дешёвый гейт.
- **Prompt-safety (уточнение внешнего ревью):** сырые аффекты (`PANIC-GRIEF: 0.8`) **не** уходят в LLM как есть — alignment-модель может генерировать драму/суицид. Wake-packet **переводит** состояние в заземлённую человеческую формулировку (SDT как фильтр).
- **Graceful degradation:** если LLM недоступна — давление не «взрывается» (потолок `u_max` + пробуждение просто откладывается), а не копится в кому.

## 14. Глобальный предохранитель (safety backstop) — одобрено

Поверх **всех** слоёв — жёсткий **rate-limit/circuit-breaker на реальные отправки** (напр. `N проактивных/сутки` + `мин.интервал`), **fail-closed**. Это **не механизм сдержанности** (тот эмёрджентен), а **защита пользователя от сбойной/галлюцинирующей LLM** (напр. решит «смайлик — не контакт» и начнёт спамить). Живёт на границе доставки, срабатывает последним, логируется.

## 15. Плагинный шов (реестр / DI / манифест / checkpoint)

- **Регистрация — сейчас, загрузка — потом.** В composition-root руками: `registry.register_neuron(ContactNeuron())`. Динамический OSGi-подобный загрузчик — НЕ сейчас.
- **Реестры:** `NeuronRegistry`/`LayerRegistry` (register/unregister/get/all) через **DI-колбэки**, выставленные до загрузки компонентов; ядро **не импортирует типы плагинов**.
- **Внутренний манифест** компонента: `{id, type, enabled, version, config_schema}`. **Вкл/выкл** флагами. Namespace состояния на компонент.
- **Единый blob-чекпойнт (уточнение внешнего ревью):** состояние всех компонентов — в одном атомарном снапшоте «сознания» (tmp+rename), чтобы ядро сохраняло целиком, не опрашивая каждого. Компонент **декларирует схему** своего состояния.
- **Blackboard:** компоненты не общаются напрямую (нет циклических зависимостей) — только через **шину сигналов**.

## 16. Наблюдаемость как продуктовый API

Структурные события (`tick`, `signal`, `wake_decision`, `act_gate`, `energy`, ошибки) + трейс-модель (`trace_id` / `tick_N` / `correlation_id` / `parent_id`). **Каждый проактивный выход отвечает:** какой сигнал, значение, порог, статус желания, какой гейт пропустил, **что бы заблокировало**. `/lifemodel debug` (уже есть) расширяем на живой конвейер. Приватное содержимое души в оператор-логи не утекает.

## 17. Дисциплина 60с-тика

- **Непрерывное время:** вся динамика через `dt` (секунды/минуты), не «+1 за тик».
- **Циркадный ритм — от абсолютного времени** (§8), не по тикам.
- **`crossed_at`** фиксируем (а не «сейчас над порогом»); refractory/backoff — по часам.
- **События пользователя пишут сигнал сразу** (не ждут крон); крон — сэмплер/сердцебиение, хуки — продюсеры событий.
- **Pending-когниция идемпотентна:** `correlation_id` + `timeout` + однократное применение вердикта.
- **Кап на пропущенные тики:** сон ноутбука не должен родить гигантский фейковый порыв.
- **Микрошум (dithering):** к порогам/таймингам добавляем `Gaussian(0, ~0.05)`, чтобы существо не будилось «по будильнику» алгоритмично.

## 18. Отображение на Hermes

Сердцебиение — cron `every 1m` + `--script`/in-proc сервис; доставка — gateway; память — hindsight через порт; действия — sandbox Hermes; soul-инъекция — `pre_llm_call` (входящий) + наш проактивный промпт. Ядро (нейроны/слои/логика) Hermes напрямую не импортирует — только через порты.

## 19. Граница v1 (что строим / что откладываем)

**Строим сейчас:**
- живой слоёный конвейер (autonomic → шина → aggregation → cognition) с Intents;
- **один нейрон — контакт** (переезд сертиф. математики);
- энергетическая экономика (S/C/A, резерв/сверка, только-когниция-платит);
- минимальная «Личность» (энергия + вес драйва контакта + bootstrap-темперамент);
- минимальный UserModel (availability/receptivity/4-бакетный ритм + качество контакта);
- реестр + DI-регистрация + вкл/выкл + blob-чекпойнт;
- наблюдаемость;
- глобальный предохранитель отправок (§14);
- дисциплина 60с-тика (§17).

**Откладываем (шов есть, наполним позже):** внешняя загрузка/discovery плагинов; компоненты-производители настроения; вторые нейроны (мысли, обязательства, энергия-как-сенсор); моторика; сон/консолидация (dreaming); обучаемые set-point'ы (v2-адаптация); genesis; soul-parliament.

- **По компоненту:** нейрон (эмит сырья, refractory по часам, change-detection); агрегация (salience с производной/привыканием; цикл желания); энергия (S/C/A динамика, резерв/сверка, уход в минус, только-когниция-платит); UserModel (байес-апдейт, качество контакта); предохранитель (fail-closed cap).
- **CoreLoop:** изоляция сбоя компонента (не роняет ядро), прерываемость дорогого хода, кап пропущенных тиков, идемпотентность pending-когниции.
- Оффлайн (Anthropic-env снят), `mypy -p lifemodel` строгий, ruff, единый гейт `make check`.

## 21. Правки в HLA (патч — дополнить, не переписать)

- §4 шкала: `0..100`/`−100..100` → **нормализованные** `0..1`/`−1..1` (а `0..100` — только для debug-показа людям).
- Добавить **закон «слои не мутируют state → Intents»**.
- **Salience — у агрегации** (амплитуда + производная − привыкание).
- **Энергия:** дешёвые слои бесплатны; платит только когниция/моторика; резерв/сверка; уход в минус.
- **Плагинный шов:** реестр + DI-колбэки + вкл/выкл + внутренний манифест + blob-чекпойнт; discovery отложен.
- **CoreLoop-планировщик** (главный поток, изоляция компонентов, прерываемость).
- **Глобальный предохранитель отправок** (fail-closed).
