## 1. Контекст и цель

`lifemodel` — проактивный агент как **«цифровой человек»**: тянется к человеку сам, когда копится нужда. Метафора — **тело как инженерное ограничение** (энергосбережение + слоёная обработка).

**Проблема.** Живая реализация выродилась в **монолит одного драйва «контакт»** (`core/decision.py`), обошедший слои. Дрейф от HLA.

**Цель.** Пересобрать **настоящее слоёное ядро** (AUTONOMIC → AGGREGATION → COGNITION с живой шиной, единым владельцем состояния, планировщиком и энергией), переселив контакт-драйв **первым нейроном внутрь слоёв**. Ядро — чистое, DI, компоненты-плагины, расширяемо без переписывания.

## 2. Принципы (законы)

1. **Логика — в слоях, не в нейроне.** Нейрон измеряет и эмитит; решает слой выше.
2. **Ниже = глупее и дешевле.** Смысл слоёв — экономить энергию, включая дорогое по порогу.
3. **Единый владелец состояния `[R]`.** Модельное состояние мутирует **только state-actor** (§7.1). Все остальные (нейроны, слои, хуки Hermes, крон) лишь **кладут события/интенты** в очередь. Никаких прямых записей — это убирает гонки многих писателей (противоречие HLA §9).
4. **Слои не мутируют состояние.** Возвращают **Intents**; применяет их state-actor атомарно в конце тика (§6).
5. **Отправка ≠ контакт `[R]`.** Проактивная отправка — это НЕ контакт: она ставит `ActionPending` (§9.2), но **не насыщает** драйв. Насыщает только **реальная вовлечённость пользователя** (§10).
6. **Сдержанность эмёрджентна.** Тишина по умолчанию; говорим только когда эффективное давление перешло порог И когниция сочла повод. Инварианты безопасности — **как тесты** (§12), плюс жёсткий предохранитель (§14).
7. **CoreLoop неубиваем.** Сердце в главном потоке; компоненты — асинхронные, изолированные (§7).
8. **Всё числовое — ограничено и воспроизводимо `[R]`.** Нормализованные величины клампятся; стохастика фиксируется сидом на тик (§5, §17). Агенты падают именно на несходящихся числах в фоне сутками.
9. **DI + гексагон;** ядро не импортирует типы плагинов. **fail-open для сенсоров, fail-closed для безопасности.**

## 3. Слоёный конвейер

```
Каналы → Сигналы(durable шина) → AUTONOMIC(нейроны,0 LLM) → AGGREGATION(0 LLM) → COGNITION(LLM) → Intents → state-actor применяет + checkpoint
```
- **AUTONOMIC** — нейроны, тупые сенсоры. **0 энергии, каждый тик.**
- **AGGREGATION** — шина, **salience** (§5), habituation, жизненный цикл желания (§12), порог. **0 энергии, каждый тик.**
- **COGNITION** — LLM (fast→smart); **act-gate**. **Единственный потребитель энергии** (§8).

> `[R]` **Ловушка комы:** энергия гасит **только** дорогое (когниция). Дешёвые слои бесплатны и всегда on — иначе при нуле заряда существо не заметит пробуждения.

## 4. Сигналы и шкала

- **Шкала нормализованная типизированная:** **униполярно `0..1`** (драйвы/дефициты), **биполярно `−1..1`** (валентные: настроение/receptivity).
- **Нейрон эмитит только сырьё:** `{value, delta, duration_over_threshold, confidence}`.
- **`priority` — НЕ срочность от нейрона `[R]`.** Это статическая метаданная **вида** сигнала в таксономии (класс), не вычисляемая нейроном важность. Важность = salience (§5), считает агрегация.
- **Реестр таксономии:** каждый вид объявляет `{schema, polarity, unit/normalization, ttl, privacy}`.
- **Шина сигналов — durable append-only** (отдельно от state-blob, см. §15). Дедуп по `origin-id` с TTL.
- Форма: `id, kind, source, polarity, value, metrics{delta, rate_of_change, confidence}, priority(static), ts, ttl, correlation_id, parent_id`.

## 5. Salience — считает агрегация, ограниченная и воспроизводимая

Важность требует контекста всех сигналов → работа агрегации.

`[R]` **Числовые правила (обязательны):**
- `rate_of_change` — от **сглаженного окна (EMA)**, НЕ от мгновенного `dt` (async-хуки в мс дают взрыв производной). Пол `dt_min`.
- Нормализация значения — **по виду сигнала** (из таксономии), чтобы разнотипные не смешивались в сыром `value`.
- **Кламп в `[0,1]`:** `salience = clamp(w1·value + w2·ema_rate − w3·habituation, 0, 1)`. Act-gate ожидает нормализованный вход.
- `baseline = EMA(value)`; привыкание — отклонение от baseline (Weber-Fechner). Salience — общая «валюта сравнения».

### 5.1. Backpressure — агрегация не падает под потоком `[R2]`

Биологический принцип: при потоке «орущих» сенсоров агрегация **сбрасывает нагрузку**, но **никогда не отказывает** (как организм при сверх-боли гасит сознание, а не ломается). Слой обязан быть **O(ограниченного) на тик** — объём входа НЕ влияет на то, отработает ли тик.

`[R2]` **Сброс — по семантике потери, НЕ по salience глобально (Codex).** Salience как единая политика сброса опасна — может выкинуть управляющее событие. Сначала классифицируем сигнал по потере:
- **Control-полоса (`lossless`)** — `exchange`, `verdict`, `delivery_result`, `in_flight`, safety/lifecycle. **Обрабатывается ПЕРВОЙ, никогда не sheddится по важности.** Ограничивается коалесингом/компакцией (`in_flight` → latest; повторы → свежайший), а при переполнении **не двигаем offset шины** — хвост остаётся на следующий тик (не теряем и не пухнем в памяти). `MAX_CONTROL` — безопасный префикс.
- **Sensor-полоса (`latest-wins`/`aggregate`/`droppable`)** — `contact`, будущие ping/presence. Коалесинг в пер-`kind` резервуар (`latest, max, count, first_ts, last_ts` → один агрегат на `kind`/тик); при `pending > high_watermark` **shed по salience** до `MAX_SENSOR`, режим до `low_watermark` (гистерезис). **Вот здесь** «сброс по важности» уместен.
- **Реальные сообщения пользователя агрегацией не «шеддятся»:** обрабатываем ограниченный **префикс**, offset для остатка **не двигаем** (backpressure = не-потребление; верхняя граница — очередь транспорта, не наш слой).
- **Стык с арузалом:** затяжная перегрузка поднимает арузал за пик Йеркса-Додсона (§8) → выше эффективный порог / тусклее когниция («слишком больно → тише сознание»).
- **Инвариант «слой не падает» (тест):** `N ≫ MAX_*` сигналов в один тик → тик за ограниченное время; control не потерян; sensor сколлапсирован/сброшен; исключений нет.

`[R2]` **v1 (один сенсор): скелет полос сейчас, salience «скучный» (Codex).** Полноценный salience (Weber-Fechner + арузал) избыточен, пока сенсор один. Строим **классификацию полос** (control lossless / sensor latest-wins) — она и защищает слой; salience-веса тривиальны до появления ping/presence/мыслей.

## 6. Intents (единственный путь мутаций)

Слои и хуки **не меняют state сами** — возвращают/кладут интенты; **state-actor применяет их батчем атомарно** в конце тика.
- Набор v1: `UPDATE_STATE`, `EMIT_SIGNAL`, `ACK_DESIRE`, `DEFER_DESIRE`, `WAKE_COGNITION`, `SEND_MESSAGE`, `RESERVE_ENERGY`/`COMMIT_ENERGY`, `UPDATE_USER_MODEL`, **`CHECKPOINT_STATE`**.
- `[R]` **Checkpoint — это интент**, который state-actor генерирует сам в конце тика, если были мутации. Убирает гонку «кто-то дёрнул save_state() посреди применения батча».

## 7. CoreLoop — сердце, планировщик, консистентность

### 7.1. Единый state-actor `[R]`
- Единственный, кто мутирует модельное состояние. Крутится **последовательно** (один поток исполнения логики).
- **Продюсеры событий** (нейроны, хуки Hermes, входящие сообщения) **кладут события/интенты в очередь**, состояние не трогают. Адаптеры Hermes — тоже только enqueue.
- В конце тика: слить очередь → применить интенты в памяти атомарно → `CHECKPOINT_STATE`.

### 7.2. Планировщик и изоляция
- Дешёвые слои — каждый тик; дорогие (когниция) — **по бюджету энергии** (§8).
- **Главный поток неубиваем:** каждый вызов компонента обёрнут; исключение → компонент пропущен; повтор → circuit-breaker («живёт без органа»).
- **Компоненты — асинхронные, изолированные;** даже теоретически не роняют ядро.

### 7.3. Асинхронная когниция через тики `[R]` (must-fix)
LLM-ход длится дольше тика, а состояние под ним меняется. При пробуждении фиксируем **контекст-версию**: `{desire_id, desire_version, state_version, reservation_id, deadline}`.
- **Семантическая инвалидация, не версионная `[R]`.** НЕ роняем вердикт на любой инкремент глобального `state_version` (тик энергии/настроения не должен убивать годный ответ — хрупко). Вердикт протух **только если семантически неуместен**: (а) желание уже разрешено (напр. пользователь ответил → `observe_exchange`); (б) `effective_pressure` упало ниже порога (нужда удовлетворена, пока думали); (в) появилось новое сообщение пользователя после пробуждения — проверяем **курсор** `last_user_message_offset`, а не абстрактную версию; (г) вышел `deadline`. Протух → отбросить (энергию сверить по факту, остаток вернуть; реактивный ход при этом уже ответил пользователю — двойного сообщения нет).
- Иначе — применить как интенты через state-actor. Применение вердикта **идемпотентно** по `desire_id`.
- **Прерываемость:** реальное сообщение пользователя = рефлекторная дуга; ядро вправе отменить/перепланировать дорогой ход (частично уже делает FIFO Hermes).

### 7.4. События сразу vs атомарность `[R]`
Разрешение противоречия «§17 пишет сразу» ↔ «§6 применяет в конце тика»: **немедленные продюсеры лишь дописывают неизменяемые события в шину/лог**; производное состояние мутирует только state-actor в свой тик. Так «сразу видно событие» и «состояние консистентно» не конфликтуют.

### 7.5. Единый планировщик `[R]`
**In-proc сервис — единственный «мозг»** (владеет тиком/решениями). **Крон — идемпотентный сторож:** тикает только если сервис мёртв (liveness-штамп), проактивно **никогда не решает**. Двойных тиков/чекпойнтов/писателей нет.

## 8. Энергия — физиология + бюджет

Модель: **двухпроцессная (Борбели)** + **арузал (Йеркс-Додсон)**. Ego-depletion не моделируем.
- **S — долг усталости** `[0..1]`: растёт от бодрствования/работы, гасится покоем.
- **C — циркадный ритм:** функция **абсолютного времени** (Unix ts), не по тикам: `C(t)=0.5+0.5·cos(2π·(t−φ)/24h)`.
- **A — арузал** `[0..1]` (перевёрнутая-U).
- **E — энергия/бюджет** `[…1]` (может уходить в минус).

**Роли (без двойного учёта) `[R]`:** **E — расходуемый бюджет**; из него списывается цена дорогих ходов. **S — не второй бюджет, а множитель цены:** усталость снижает КПД → `cost_real = cost_base·(1 + α·S)`. Так «сел и устал → думать дороже → сам уходит в рефлексы» — эмёрджентно, без `if`.
- Дешёвые слои = **0** в энергоединицах.
- **Прайс-лист** — v1 фикс (bootstrap-константы, не финальная физиология): `fast_llm≈0.02`, `smart_llm≈0.08`, `send≈0.03`.

**Жизненный цикл резервации `[R]`:**
- `reserve(estimate)` перед дорогим ходом (гейт «хватает ли `E ≥ estimate·(1+αS)`»).
- По завершении: `commit(actual)` + `refund(estimate − actual)`.
- **Отмена/таймаут/сбой доставки/блок предохранителем:** списываем **фактически потраченное** (токены до отмены), остальное возвращаем. Reservation-id связывает резерв с ходом.

`[R]` **Инвариант «не окаменеть».** Цена самого дешёвого хода при максимальной усталости `cost_fast·(1+α·S_max)` должна быть **строго меньше** зарезервированного минимума `E` — чтобы существо всегда могло сделать хотя бы один минимальный ход (пожаловаться на усталость / дойти до сна), а не застыть без энергии обработать act-gate. `α` калибруется под это.

## 9. Личность (self-model)

Три вида характеристик: **темперамент** (конфиг: веса драйвов, ёмкость энергии, восстановление, чувствительности + **bootstrap** для cold-start), **быстрые состояния** (энергия, S, арузал, настроение, давление), **медленно-выученное** (set-point'ы — позже).

**Движок драйвов — Панксепп** (SEEKING/CARE/PLAY/FEAR/RAGE/PANIC-GRIEF). Контакт = **гомеостат сепарационного дистресса (PANIC/GRIEF)**. **SDT** — язык/фильтр (§13), не движок. **Настроение — Рассел** (валентность×арузал), заземлено в сигналах, фактов не выдумывает.

### 9.1. Давление: латентное vs эффективное `[R]` (must-fix)
Три поля вместо одного:
- **`latent_pressure`** — истинная депривация: растёт по **реальному wall-clock одиночеству**. Питает **эскалацию/`duration_over_threshold`** (анти-заброс).
- **`inhibition`** — фактор подавления `[0..1]` (см. 9.2).
- **`effective_pressure = latent · (1 − inhibition)`** — питает **порог пробуждения**.

### 9.2. ActionPending с затуханием `[R]` (устраняет «ghosting»)
После отправки ставим `inhibition = I0`. `[R]` **Grace-period, потом затухание** (иначе «ghosting 2.0»: экспонента с 1-й секунды + высокое `latent` могут пробить порог через 15 мин → спам). Формула:
```
I(t) = I0,                       t ≤ T_grace     (жёсткое подавление — гарантированное окно тишины 30–60 мин)
I(t) = I0·e^(−λ·(t − T_grace)),  t > T_grace      (плавный распад ~часы)
```
Даём человеку железобетонное окно на ответ **независимо** от величины скрытого одиночества. Пока ждём — эффективное давление подавлено. **Проигнорировали часами → подавление истаяло → латентное одиночество бьёт обратно** (не «написал и забыл навсегда»). `latent` растёт всегда — обучаемые set-point'ы будущего тренируются на **латентном**, не на подавленном (иначе отравятся).

`[R2]` **Обвязка ActionPending (Codex):** `inhibition = I0` ставим **только после подтверждённой доставки** реального проактивного сообщения — НЕ на пробуждение, НЕ на fulfill от LLM, НЕ на fulfill, заблокированный предохранителем. `[SILENT]`/`REJECT` → только короткий cognition-backoff, **не** inhibition. Провал доставки — не насыщает и не даёт grace-плато; максимум короткий retry-backoff. Латентное = «правда», эффективное = «разрешение проснуться»: эскалация от высокого `latent` поднимает срочность/контекст, но **не обходит** мин.интервал отправки (§14).

## 10. UserModel (модель пользователя)

Байесовская оценка **с неопределённостью** из наблюдаемого: `availability [0..1]`, `receptivity [−1..1]`, `rhythm` — **4 бакета** (утро/день/вечер/ночь).
- **Качество контакта, не только скорость `[R]`:** быстрый «ок» ≠ успех. Метрика вовлечённости (напр. `tokens_user/tokens_agent`) в агрегации.
- **Насыщение драйва — только на реальную вовлечённость** (§5 принципов): именно `observe_exchange` с положительным качеством гасит `latent`, а не факт нашей отправки.
- Выдаёт **уместность** = `salience × availability × norm(receptivity) × обратная_цена_прерывания` — вход act-gate.

## 11. Нейроны (контракт + ContactNeuron)

- Контракт: `check(state, now, correlation_id) -> Signal | None`. Тупой сенсор.
- BaseNeuron: refractory **по часам**; change-detection (Weber-Fechner); `last_value`.
- **ContactNeuron (v1):** переселяет сертифицированную математику драйва; питает `latent_pressure` (рост по wall-clock тишине, satiate — из §10). Backoff/дедуп внутри нет — это агрегация.

## 12. Агрегация (жизненный цикл желания + инварианты)

Желание — объект с циклом: рождение (первый переход **порога по `effective_pressure`**) → дедуп/`ack` → вердикт → release отложенного.
- **Вердикт `[R]`:** «исполнить» = отправить; `ActionPending` ставится **только после подтверждённой доставки** (§9.2), давление **не сбрасывать** (насыщение — от реального ответа); «отложить» = держим; «отвергнуть» = убрать + короткий backoff.
- **`duration_over_threshold` считается на `latent`** (§9.1) — иначе игнор навсегда блокирует эскалацию.
- `[R2]` **Идентичность желания (Codex):** у желания есть `desire_id`. **`WAKE_COGNITION(desire_id)` эмитится ТОЛЬКО на переходе** NONE→ACTIVE / DEFERRED→ACTIVE — не выводится из `status==active` (иначе дедуп-урж ложно будит когницию). Вердикт **несёт `desire_id` и идемпотентен**: применяется, только если ссылается на **живое** желание; устаревший (желание уже очищено/сменилось на новое) — **no-op**. В одном тике **exchange доминирует над verdict**: реальный ответ чистит желание, рассинхронный fulfill игнорируется.

**Инварианты как ТЕСТЫ:** одно желание на переход; дедуп живого; «отложить» не сбрасывает; «отвергнуть»→backoff; **насыщает только реальная вовлечённость**; свой импульс ≠ контакт; **пробуждение ≠ отправка**; `[R2]` **`WAKE_COGNITION` только на переходе (не на дедуп-урж)**; **вердикт применяется лишь к живому `desire_id`, устаревший — no-op**; **exchange доминирует над verdict в тике**; **control-сигналы никогда не sheddятся** (§5.1); **offset шины двигается только для реально включённых/безопасно сжатых сигналов**; **`ActionPending` — только после подтверждённой доставки** (§9.2); **`[SILENT]` — успешное разрешение, не попытка доставки**. Сертифицированные сценарии старого драйва переносятся на границу слоёв.

## 13. Когниция (wake-packet, act-gate, prompt-safety)

- **Wake-packet** = soul + повод (сигнал/значение/порог, состояние, уместность из UserModel).
- **Act-gate:** исполнить / отложить-и-держать / отвергнуть (`[SILENT]`).
- **Prompt-safety `[R]`:** сырые аффекты (`PANIC-GRIEF:0.8`) не уходят в LLM. **Таблица проекций:** на (драйв × полоса значений) — небольшой набор **синонимичных** человеческих формулировок; выбор псевдослучайный с **сидом от `correlation_id`**; логируем `projection_id`. **Безопасностно-значимый смысл не рандомизируем** — только стиль/формулировку (чтобы преамбула не была детерминированной и ответы не были однообразны).
- **Никакого «таймерного» самоповествования `[R2]`.** Wake-packet несёт **желание**, а не причину-часы: в LLM уходит «тянет на контакт / соскучился», НЕ «прошло N часов тишины». Существо владеет своим «захотел» и не цитирует механизм. (Наблюдённый анти-паттерн старого монолита: сообщения вида «шесть часов тишины», «три часа назад проверял» — существо объясняет себя будильником.) Сырое время/`dt` в промпт **не попадает** — только проекция §13.
- **Тишина вместо пустого выхода `[R2]`.** Нет реального повода/мысли → act-gate возвращает `[SILENT]`, а НЕ сообщение «мне нечего сказать» (тот же анти-паттерн монолита, у которого act-gate отсутствовал: проснулся → отправил шаблон). Проактив органичен, когда есть мысль (придёт со вторыми нейронами — мысли/обязательства); при одном контакт-нейроне легитимный повод — искренняя нужда контакта, не расписание.
- `[R2]` **Инфо-барьеры, не только проекция (Codex).** Проекции мало: если LLM видит таймстемпы истории — сам пересчитает «6 часов». Из проактивного контекста **вырезаем/бакетируем точное время** (порядок событий — да, точные длительности — нет; «давно было тихо», не «N часов»); правило промпта «не ссылайся на часы/таймеры/пороги/проверки/мониторинг как причину». **Output-lint перед отправкой:** режем/переписываем сообщения со словами «часов/минут/проверял/таймер/тишина N/нечего сказать». Пакет несёт `desire_frame` / `human_motive` / `allowed_self_narrative`; сырая механика — только оператор-лог.
- **Graceful degradation:** LLM недоступна → давление не «взрывается» (потолок + пробуждение откладывается).

## 14. Глобальный предохранитель (safety backstop)

Поверх всех слоёв — жёсткий **rate-limit на реальные отправки** (`N/сутки` + мин.интервал), **fail-closed**. Защита пользователя от сбойной/галлюцинирующей LLM, не механизм сдержанности.
- `[R]` **Исход при блокировке:** если когниция сказала «исполнить», а лимит заблокировал — желание **НЕ помечаем fulfilled**; `[R2]` **желание держим живым** (defer/hold, **не чистим** — чтобы дослать позже), **давление сохраняем**, `ActionPending` **не ставим** (доставки не было — §9.2). Логируем причину блока.
- `[R2]` **`[SILENT]` ≠ доставка (Codex):** `[SILENT]`/`REJECT` — это **успешное разрешение когниции**, а не попытка отправки: `ActionPending` не ставится, backstop-счётчик реальных отправок не трогается.

## 15. Состояние: владелец, чекпойнт, шина `[R]`

- **Один state-blob** — модельное состояние всех компонентов (namespace на компонент; компонент декларирует схему). Пишется **только** через `CHECKPOINT_STATE` (§6), атомарно (tmp+rename).
- **Шина сигналов — отдельный durable append-only лог.** Чтобы replay не дублировал/не терял сигналы, **потреблённый offset шины хранится ВНУТРИ state-blob** → один атомарный чекпойнт фиксирует и состояние, и «докуда потребили шину». Коммитятся вместе.
- `[R]` **Обрезка шины (иначе лог растёт в гигабайты).** Hermes сыплет сотни микро-событий (ping/presence). Интент/крон `PRUNE_BUS` (напр. каждые N тиков): по надёжно закоммиченному `committed_offset` из последнего blob безопасно обрезать записи `offset < committed_offset` (или bounded-ring). Резать только ниже закоммиченного — replay не пострадает.
- **Blackboard:** компоненты общаются только через шину, не напрямую (нет циклов).
- **Плагинный шов:** реестры + DI-колбэки, вкл/выкл, внутренний манифест `{id,type,enabled,version,config}`. **Регистрация — сейчас, внешняя загрузка/discovery — потом.**

## 16. Наблюдаемость как продуктовый API `[R]`

Структурные события + трейс (`trace_id`/`tick`/`correlation_id`/`parent_id`). Каждый проактивный выход и каждый тик отвечают, включая (расширено по ревью):
- `state_version` до/после, `checkpoint_id`, применённые/**отброшенные протухшие** интенты;
- `latent`/`effective` давление, `inhibition`, решение дедупа;
- эффективный порог + `noise_seed`;
- энергия: reserve/actual/refund, причина cancel/timeout;
- причина блока предохранителем; `projection_id` преамбулы.
Приватное содержимое души в оператор-логи не утекает. Инструмент — `/lifemodel debug`.

## 17. Дисциплина 60с-тика `[R]`

- **Два времени раздельно:** `dt_dynamics = min(elapsed, dt_cap)` — для интеграции значений (гасит фейковый скачок после сна ноутбука); `elapsed_wall_time` — реальное для **депривации/`latent`** (отсутствие по стенным часам реально).
- Циркадный ритм — от абсолютного времени (§8).
- `crossed_at` фиксируем; refractory/backoff — по часам.
- События пользователя пишут сигнал сразу (в шину); state мутирует state-actor (§7.4).
- Pending-когниция идемпотентна (§7.3).
- **Микрошум (dithering):** сэмплим **один раз на тик** (сид `trace_id`), **клампим** (`max(ε, порог+шум)`, не уводим в минус/за 1), **логируем** (raw/шум/эффективный). Заморожен на весь прогон пайплайна — иначе агрегация и когниция увидят разные пороги.

## 18. Отображение на Hermes

Сердцебиение — in-proc сервис (авторитетный, §7.5) + крон-сторож; доставка — gateway; память — hindsight через порт; действия — sandbox; soul-инъекция — `pre_llm_call` + проактивный промпт. **Адаптеры Hermes только enqueue-ят события/интенты, модельное состояние не мутируют.**

## 19. Граница v1

**Строим:** живой конвейер (autonomic→шина→aggregation→cognition) + Intents + **state-actor/checkpoint** + **один нейрон (контакт)** с латентным/эффективным давлением и ActionPending-затуханием + **salience + backpressure агрегации (§5, §5.1: коалесинг, shed-по-важности с гистерезисом, инвариант «слой не падает»)** + энергоэкономика (S/C/A, резерв/сверка, cost·(1+αS)) + минимальная «Личность» + минимальный UserModel (+качество контакта) + реестр/DI/вкл-выкл + наблюдаемость (§16) + глобальный предохранитель + дисциплина 60с-тика + асинхронный жизненный цикл когниции (§7.3).

**Откладываем (шов есть):** внешняя загрузка плагинов; компоненты-производители настроения; вторые нейроны (мысли/обязательства/энергия-сенсор); моторика; сон/консолидация; обучаемые set-point'ы; genesis; parliament.

## 20. Тестирование (TDD)

- Перенос сертифицированных сценариев старого драйва на границу слоёв (инварианты §12 зелёные).
- По компоненту: нейрон; агрегация (salience: кламп/EMA-производная/привыкание; цикл желания; **duration на latent**); энергия (S/C/A, резерв/commit/refund, cost·(1+αS), уход в минус, отмена, **инвариант «не окаменеть»**); UserModel (байес + качество); предохранитель (fail-closed, исход-не-fulfilled); **ActionPending: grace-плато держит тишину даже при высоком latent; после T_grace распад; игнор часами → возврат латентного**.
- CoreLoop/консистентность: изоляция сбоя; **асинхронный вердикт — семантическая инвалидация** (пользователь ответил во время «размышления» → проактив отброшен, реактивный ход ответил, двойного сообщения нет; тик энергии/настроения НЕ роняет вердикт); идемпотентность по `desire_id`; **checkpoint+offset шины атомарны** (replay без дублей); **`PRUNE_BUS` режет только ниже `committed_offset`**; dithering **воспроизводим** (один сид на тик).
- Оффлайн (Anthropic-env снят); строгий `mypy -p lifemodel`; `make check`.

## 21. Правки в HLA (патч — дополнить)

- §4 шкала → нормализованные `0..1`/`−1..1` (`0..100` — только debug-показ).
- Закон **«единый state-actor; слои/хуки только через интенты»** (заменяет «много писателей под локом»).
- **Salience у агрегации**, ограниченная/воспроизводимая.
- **Энергия:** только когниция платит; `E` бюджет, `S` множитель цены; резерв/сверка/refund.
- **Давление латентное vs эффективное; ActionPending-затухание; отправка ≠ контакт.**
- **Плагинный шов** (реестр+DI, blob-checkpoint, offset шины внутри blob); discovery отложен.
- **Единый планировщик** (in-proc мозг, крон-сторож).
- **Глобальный предохранитель** (fail-closed, исход-не-fulfilled).
