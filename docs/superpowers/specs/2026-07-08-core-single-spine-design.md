# Ядро: единая спина (single-spine) — дизайн ПЕРЕСБОРКИ

**Дата:** 2026-07-08 · **Статус:** зафиксировано, строим · **Версия:** v1.0 (rebuild)
**Родословная:** конкретизирует `docs/hla.md` §1 / §4.1 / D9. Ревью: codex (независимый, с доступом к репо — проверял утверждения по коду). Высокоуровневый визуальный обзор процесса — артефакт `core-rebuild-overview`.

> **Что изменилось относительно предыдущей записи (v0.9).** Прежняя рамка — «инкрементально раствори `ThoughtGeneration` в агрегацию, контакт закрепи parity-тестами перед заменой». Она **отменена**. Причина: живая сущность **ни разу** не написала владельцу — каждый проактивный ход кончается `[SILENT]`. Parity замораживает текущее поведение как эталон → бетонирует сам баг («всегда молчит»). Новая рамка — **пересборка поведенческого слоя** до минимума, который реально функционирует и полностью наблюдаем, при **сохранении** storage/спины-фундамента (D7/D8). Контакт — первый щуп, не святыня. Инвариант D9 «`sim/` не трогаем» и HLA §2.1/§4 (нейрон копит давление) — **сознательно пересматриваются** (см. §6, §9).

## 1. Проблема (что чиним и почему по-настоящему)

Живая сущность на `~/.hermes` не инициирует контакт: проактивные ходы происходят, но каждый → `[SILENT]`. При этом «в симуляции контакт сертифицирован». Разрыв — структурный, не калибровочная случайность. Три слоя причины (подтверждены codex по коду):

1. **`sim/` сертифицирует не тот код.** `sim/harness.py::run` — **третья, отдельная реализация тика**: своя петля, игрушечный `sim/aggregation.py::Aggregator` (3 состояния), cognition как FIFO scripted-вердиктов. Живой путь другой: `core/coreloop.py` гоняет `contact_neuron → thought_crystallization → aggregation → thought_attention → thought_generation → cognition` (`composition.py:209`) и коммитит через `StateActor`. Между «urge по мат-модели» и «реально отправлено» вживую стоит ~6 гейтов, которых в sim НЕТ. Значит «сертифицировано» ≠ «вживую заговорит» **by construction**.

2. **Сам промпт учит молчать (не только калибровка).** Wake-packet буквально инструктирует: «если наполнитель — ответь `[SILENT]`» (`core/wake_packet.py:39`) и «не выдумывай повод» (`:81`). Если на входе только «тихая тяга» (`core/projection.py:33`), ход **закономерно** молчит. Плюс аварийный клапан «острая тоска = повод» **недостижим по калибровке**: генерация даёт salience `_EVENT_DRIVE_SALIENCE = 0.45` (`core/thought_generation.py:100`), а пол own-longing — `STRONG_LONGING_SALIENCE = 0.75` (`core/thought_crystallization.py:66`). Ветка живьём не берётся (`lm-wuf`).

3. **Барочность → неотлаживаемость и ложь в observability.** Два независимых механизма рождения (`ContactAggregation` из нейрона + отдельный `ThoughtGeneration` со своими idle/chaining-триггерами) дают дубли/спираль тишины. Шесть гейтов подавления (receptivity-veto, effective_pressure/inhibition, pure_longing_repeat_hold, energy-affordability, jitter ~20%, backstop) схлопнуты в один ложный исход `skipped_busy`: `ReachOutcome.SKIPPED_BUSY` рождён как «сессия занята, повтори» (`domain/egress.py:18`), но переназначен на «пайплайн не родил launch» (`core/proactive.py:60-62,68`). Модели сессии/агента у нас нет → «busy» без референта → причина молчания невидима.

**Отдельно (codex):** `ReachOutcome.DELIVERED` ≠ «пользователь увидел текст». `gateway_core.inject_proactive_turn()` (`gateway_core.py:132`) возвращает delivered после **постановки** internal-хода в очередь; реальный исход «написал / промолчал» приходит позже в `post_llm_call` как `proactive_outcome` (`hooks.py:199`).

## 2. Рамка ребилда (что делаем и чего НЕ делаем)

**Пересобираем поведенческий слой** (сенсор → драйв → агрегация → когниция/act-gate → egress) до **минимума, который реально шлёт сообщение и полностью наблюдаем**. **Оставляем** storage/спину-фундамент (тик `CoreLoop`, intent-шина + атомарный коммиттер, один SQLite + три порта, трейс-контекст — D7/D8): разрыв не в нём.

Управляющий закон (BRD/HLA), который код нарушил и который теперь ведёт ребилд:
- §9.2 «стало барочно — спроси *а у людей так?* и режь»; §9.4 «ничего невидимого»; NFR2 «личность = проза + **минимум** машинерии»; roadmap «сначала движок на **одном** нейроне, YAGNI, механизм ядра — по нужде».

Богатую BDI-машинерию (Рубикон-кристаллизация как церемония, receptivity-appraisal, top-down thought-springs, jitter, деревья мыслей) **наперёд не тянем** — возвращаем по нужде, когда сценарий её потребует. НО: **безопасность и ненавязчивость — сохраняем** (§8; правка codex — не путать «сделать, чтобы заговорило» со «снести всё»).

## 3. Архитектура спины

Пять звеньев спины (+ внешний act-gate — асинхронный ход Hermes), сверху вниз дороже/умнее. Дешёвые холодные слои (0 LLM, 0 энергии) работают каждый тик; дорогой **ход Hermes** (LLM) запускается только за порогом — в самом core LLM синхронно не вызывается. Ничто не рождает durable-сущность вне спины. **Имена — рабочие.**

| Звено | Слой · стоимость | Ответственность (одна) | Владеет / эмитит |
|---|---|---|---|
| `PresenceNeuron` | AUTONOMIC · 0 LLM | **Мгновенный** замер контакт-канала: минут тишины + был ли обмен сейчас (сырые признаки). Не копит, не владеет, не решает. | durable-state: **нет**; эмитит сырой скаляр |
| `SolitudeDrive` (интегратор) | AUTONOMIC · 0 LLM | **Центральная интеграция** дефицита присутствия в витальный `u`; сатиация **только** на реальном обмене. | **владеет `u`** (`runtime_state`); эмитит transient `contact_pressure` |
| `ContactAggregation` | AGGREGATION · 0 LLM | Читает **свежий `u` из transient-сигнала `contact_pressure`** — это единственный same-tick путь (`UpdateState({"u":…})` от драйва виден лишь **после** коммита; см. §4). `ctx.state.u` — только persisted fallback / baseline начала тика. **НЕ** через `PressureSensorPort` (тот суммаризует desire-записи, не витальный `u`). Считает salience, прогоняет `evaluate_wake`. Порог → рождает **одно** желание (дедуп по живому). Не порог → **span подавления с reason**. | рождает **Desire** (`memory_records`) / suppression-span |
| `CognitionLauncher` | COGNITION-launch · **0 LLM в core** | Просыпается по порогу. Резервирует энергию, строит wake-packet с **переписанным контрактом** («твоя острая тоска — достаточный настоящий повод»), эмитит `LaunchProactive` (= запуск дорогого хода Hermes). **Сам LLM не вызывает.** | эмитит **Intention** + `LaunchProactive` |
| Hermes-ход = **act-gate** | внешний · LLM | Настоящий act-gate: разбуженный ход Hermes решает сказать / **промолчать (`[SILENT]`)**. Асинхронно; вердикт возвращается **read-back**'ом. | verdict → `proactive_outcome` (`post_llm_call`) |
| `proactive egress` | EGRESS · граница | Доставка через gateway. `ReachOutcome` = **только граница доставки**. Реальный исход — read-back из `post_llm_call`. | backstop; read-back `proactive_outcome` |

**Трёхчастный разрез сенсор → драйв → агрегация (правка codex, обоснование SRP).** Сейчас `contact_neuron.py:31` смешивает четыре работы: чтение часов (`dt`), интеграцию `u` (`drive.rise`), сенсинг обмена + качество (`satiate(q=…)`), владение/запись `u` («Sole writer of u»). Разносим: сенсор — только мгновенный замер; **интеграция и владение `u`** уходят в отдельный AUTONOMIC drive-компонент (а **не** в агрегацию — иначе агрегация станет и физиологией, и birth-policy сразу); агрегация читает `u` и владеет жизненным циклом желания + причинами молчания. Биология: рецептор чует мгновенное, накопленный драйв-дефицит интегрирует центр (осморецептор vs жажда).

## 4. Процесс одного тика

Тик атомарный. Никакой слой не пишет стор напрямую — только интенты + коммиттер.

1. **Снимок начала тика.** Один консистентный read: живые объекты (`memory_records`), витальное (`u`/энергия), входящие сигналы. Все компоненты видят одно и то же.
2. **Компоненты по порядку** (`PresenceNeuron → SolitudeDrive → ContactAggregation → CognitionLauncher`), **каждый — дочерний span.** Возвращают **интенты** (`UpdateState`/`PutRecord`/`TransitionRecord`/`EmitSignal`/`LaunchProactive`); состояние сами не меняют. `EmitSignal` виден последующим компонентам в этом же тике (не переопубликовывается).
3. **Один атомарный коммит.** `StateActor` применяет все интенты одной SQLite-транзакцией в конце тика. Новые сущности видны со **следующего** тика. Split-brain невозможен by construction.
4. **Egress — если родился `LaunchProactive`.** Backstop → доставка. Иначе тик закрывается тихо, но с полным деревом span'ов «почему».
5. **Read-back исхода (асинхронный act-gate).** Следующий тик читает реальный вердикт хода Hermes (`post_llm_call`): **сообщение отправлено** → ActionPending-инхибиция + очистка `pending` + send-ledger (**send ≠ contact: `u` НЕ гасится**); **`[SILENT]`** → REJECT + backoff. `u` гасит **только** genuine inbound exchange (инвариант §9.7).

**Два режима:** тихий тик (сенсор+драйв+агрегация, 0 LLM/энергии, молчит с причиной «ниже порога») vs порог перейдён (просыпается когниция, act-gate решает).

## 5. Наблюдаемость — принудительный инвариант ядра

Сильнее текущего кода (где `bound_log_context` — no-op без трейса, `coreloop.py:61-75`, а `TracerPort.child_of()` вообще не используется, `ports/tracer.py:73` → компонентных span'ов сейчас нет).

- **Лог без активного trace+span структурно невозможен** — не дисциплиной. Убираем путь «untraced».
- **Корень = единица исполнения** (тик / входящий ход): на старте минтится `trace_id`, тик — root-span. Каждое событие — дочерний span, у всех есть родитель, кроме корня.
- **«Почему НЕ произошло» — первоклассный span подавления с `reason`** (какой гейт заглушил). Прямая замена вранья `skipped_busy`. Молчание перестаёт быть отсутствием записи.
- **OTel-форма, stdlib-реализация.** W3C `traceparent`-совместимо; **без** хард-зависимости на OTel-пакет (плагин бежит в venv Hermes); экспорт — опциональный адаптер с no-op фолбэком.
- **Закрытый словарь причин подавления (контракт, не открытый вопрос).** `reason ∈ { below_threshold · in_flight · silence_window · decline_backoff · pending_proactive · energy_unaffordable · backstop_rate_limited · repeat_pure_longing · act_gate_silent · egress_unavailable · egress_failed · component_failed }` (`silence_window`/`decline_backoff` — исходы `evaluate_wake`, добавлены на T3). Каждый suppression-span несёт минимум `{reason, component, trace_id, span_id, tick}`. Enum расширяется только осознанно (новый гейт = новый код). Тесты span-дерева ассертят по этим кодам.
- **Где живёт наблюдаемость (для тестов и `/lifemodel debug`).** Источник истины — **структурные логи** (structlog, всегда включены, не зависят от опционального OTel-экспорта); последнее решение тика доступно debug-CLI (last-decision из `TickReport` / bounded-буфера). OTel-экспорт — best-effort после коммита; его отсутствие **не** должно лишать `/lifemodel debug` ответа «почему промолчал».
- Это делает sim-прогон и live-прогон **одинаково отлаживаемыми**: одно и то же дерево span'ов; видно, какой span замолчал.

## 6. Симуляция = тот же код

- **Удаляем** `sim/harness.py` (третья реализация тика) и `sim/aggregation.py` (игрушечный автомат).
- **Не выбрасываем** сертифицированную математику: live уже импортирует `evaluate_wake` из `sim/wake.py` (`aggregation.py:41`), `Drive` из `sim/drive.py` (`contact_neuron.py:15`). Вживляем её в реальные компоненты; старый `sim` понижаем до **unit-регрессий математики**.
- **Новый integration-harness** гоняет **реальный `CoreLoop` + реальные компоненты** через fake-порты: fake clock (`dt` шагами), in-memory `MemoryPort`/SQLite, recording-egress (пишет, не шлёт), scripted act-gate (вместо LLM). Ассерты — на исход **и на дерево span'ов**.
- **Инвариант:** зелёный сценарий честно предсказывает live, потому что это буквально тот же код. Это лечит класс «unit-подобный sim прошёл, конвейер — нет».
- **Разворот D9:** прежний инвариант «`sim/` не трогаем» отменён — трогаем осознанно (математику сохраняем, параллельную петлю сносим).

## 7. Лекарство от `[SILENT]` (три части, вместе)

1. **Контракт промпта.** Убрать «если наполнитель — `[SILENT]`» / «не выдумывай повод»; заменить на **«твоя собственная острая тоска — достаточный настоящий повод написать коротко»**.
2. **Достижимая калибровка.** Ветка «тоска-как-повод» должна **реально** перешагивать порог живьём, а не только в прямых тестах (снять `lm-wuf`).
3. **Молчание — сознательный выбор.** Если существо молчит — это вердикт act-gate **с залогированной причиной**, не случайный обрыв на забетонированной ветке. Тишина остаётся возможной, перестаёт быть багом по умолчанию.

**Скелет — drive-only:** острая тоски достаточно как повода. Top-down «повод-из-мысли/петли» — **по нужде** (Ф5/6): требовать его в первом щупе нечестно (промпт + калибровка почти гарантируют непорождение own-longing → снова недостижимая ветка).

## 8. Оставляем vs срезаем (safety ≠ барочность)

**Оставляем (фундамент + безопасность; упрощаем, но сохраняем концерн):**
- Атомарный коммит · один SQLite · три порта (`StatePort`/`MemoryPort`/`PressureSensorPort`) · intent-шина.
- Трейс-контекст → теперь **принудительный** (§5).
- `backstop` rate-limit (fail-closed) · `in_flight` · pending-idempotency.
- Анти-повтор pure-longing (контракт ненавязчивости) — концерн сохраняем, механику упрощаем.
- Сертифицированный `evaluate_wake` (абсолютный контакт-порог).

**Срезаем (барочность; вернём по нужде) — но только заменив честным span'ом подавления:**
- Параллельный `ThoughtGeneration` (idle / chaining → Фаза 6).
- Рубикон-кристаллизация как церемония; receptivity-appraisal (богатый гейт уместности); jitter ~20% и прочие скрытые подавители.
- Ложный `skipped_busy` → span подавления с `reason`.

**Компоненты тика — что делаем с каждым** (текущий порядок — `composition.py:209`):
- **Удаляем / отключаем:** `ThoughtCrystallization`, `ThoughtAttention`, `ThoughtGeneration` (параллельное рождение мыслей → Фаза 6).
- **Переписываем:** `ContactNeuron` → `PresenceNeuron` (мгновенный) **+** новый `SolitudeDrive` (интегратор `u`); `ContactAggregation` — срезать receptivity-appraisal / top-down thought-spring, упростить анти-повтор pure-longing, добавить suppression-span; `Cognition` → `CognitionLauncher` (честно: 0-LLM launcher, не синхронный LLM).
- **Оставляем:** `CoreLoop`, `StateActor`, порты, intent-шина, `backstop`, `energy`, вживлённые `evaluate_wake`/`Drive`, трейсер (доведённый до принудительного).
- **Новый порядок тика:** `PresenceNeuron → SolitudeDrive → ContactAggregation → CognitionLauncher`.

**Тесты и sim:**
- **Удаляем:** `sim/harness.py`, `sim/aggregation.py`; зависящие тесты (`tests/sim/test_sim_harness.py`, `tests/sim/test_sim_scenarios.py` импортируют `sim.harness.run`/`sim.aggregation.Verdict`) — переписать на новый integration-harness или удалить.
- **Оставляем как unit-регрессии математики:** тесты `sim/wake.py`/`sim/drive.py` (математика сертифицирована, не меняется).
- **Новое:** integration-harness (§6) + сценарные тесты на реальных компонентах, ассерты на исход **и** дерево span'ов.
- **Риск (явно в план):** до перекройки часть текущих зелёных тестов станет красной — это ожидаемо (меняем контракт); план обязан перечислить delete / rewrite / keep поимённо.

## 9. Инварианты (обязательны, откуда бы ни начинали)

1. **Абсолютный контакт wake.** `evaluate_wake` остаётся сертифицированным гейтом; никакой relative/Вебер-фильтр не подавляет `u ≥ θ`.
2. **Сенсоры — строго мгновенные, без durable-state.** Накопление и владение `u` — в отдельном AUTONOMIC drive-интеграторе; политика рождения желания — в агрегации. **(Пересматривает HLA §2.1/§4: давление-накопление уходит из нейрона.)**
3. **Никакой слой не пишет стор напрямую** — только intent-шина + атомарный коммиттер в конце тика (0-LLM И LLM).
4. **Наблюдаемость принудительна:** лог без span невозможен; каждое решение, включая **не-действие**, — span с причиной; у каждого события есть родитель, кроме корня (тик/ход).
5. **Симуляция гоняет реальный код.** Единственная правда поведения — реальные компоненты; sim их лишь драйвит через fake-порты. Параллельной модели тика нет.
6. **Приёмка — по реальному `proactive_outcome`** (написал текст), не по `delivered` (поставлено в очередь).
7. **`u` — витальный скаляр** (`runtime_state`), гасится **только реальным обменом** (send ≠ contact). Скрытая (не спроецированная) мысль на `u` не влияет.
8. **Хранение ≠ проекция промпта.** Проекция фильтрует silence-biased/низко-релевантное перед LLM (переносится из v0.9; актуально, как только вернутся мысли).
9. **`ReachOutcome` — только граница доставки** (`delivered / failed / unavailable`). «Почему нет сообщения» — решение ядра (span подавления), не egress-исход.
10. **Storage = единственный source-of-truth** (D7). Дедуп: сигнал потреблён-один-раз (`origin_id`) + не рождать уже живую singleton-сущность.

## 10. Приёмка

1. **sim (зелёный).** Набор сценариев на реальных компонентах через fake-порты: тишина копится → порог → повод → сообщение; **и негативы** (молчит по правильной причине; ненавязчивость держится). Каждый ассерт проверяет исход **и дерево span'ов**.
2. **live.** Деплой на `~/.hermes`; приёмка = **реальный `proactive_outcome`** из `post_llm_call` (существо действительно прислало текст владельцу).

**Риск-предохранитель (codex):** не оптимизировать приёмку под «одно сообщение любой ценой» — иначе теряем контракт ненавязчивости. Успех = **уместное** сообщение по конкретному внутреннему поводу, наблюдаемое от сенсора до текста.

## 11. Отклонённые варианты

- **Parity-first** (golden-тесты текущей `ContactAggregation` перед заменой) — замораживает «всегда молчит» как эталон; бетонирует баг. Parity сужается до узкой охраны сертифицированной математики там, где её реально переносим, — не upfront-гейт.
- **Полный снос, включая storage/порты** — больше регрессий, чем свободы; D7/D8 закрывают реальные классы split-brain. Оставляем фундамент.
- **Накопление `u` в сенсоре** — смешение периферии с центром; сенсор мгновенный, интеграция — отдельный драйв.
- **Накопление `u` в агрегации** — агрегация станет физиологией + birth-policy сразу; интеграция — отдельный AUTONOMIC-компонент.
- **Pure decision-kernel вместо real-component sim** — полезен для unit-тестов, но не заменяет end-to-end; ровно этот класс («unit прошёл, конвейер нет») и есть наш баг.
- **Top-down повод обязателен в первом щупе** — при текущей калибровке/промпте почти гарантирует недостижимую ветку.
- **Лечить `[SILENT]` одной калибровкой** — промах: сам промпт учит молчать.
