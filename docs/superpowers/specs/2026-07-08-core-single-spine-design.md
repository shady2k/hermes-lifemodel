# Ядро: единая спина рождения сущностей (single-spine) — дизайн

**Дата:** 2026-07-08 · **Статус:** зафиксировано (строим инкрементально)
**Родословная:** конкретизирует `docs/hla.md` §1–2 / §4.1. Механику событий/порогов/затухания портируем из **оригинального `lifemodel`** (github.com/shady2k/lifemodel) — там это уже решено в проде. Ревью: codex ×3 раунда + независимый архитектор (без доступа к репо) ×3 раунда, сходимость.

---

## 1. Проблема (что чиним)

Сейчас в CoreLoop **два независимых механизма рождения сущностей** работают как пиры каждый тик:
1. `ContactAggregation` — рождает контакт-желание из сигнала `ContactNeuron` (нейрон → агрегация → желание), гейт сертифицированной `sim/wake` математикой. Жёстко привязан к единственному контакт-драйву (singleton `desire:contact:owner`, контакт-специфичные поля в runtime-строке).
2. `ThoughtGeneration` — **отдельный** компонент, штампует `thought`-строки из своих триггеров (`event` / `idle`-блуждание / `chaining`-«развить предыдущую мысль») со **своим** рейт-лимитингом (энергокост, `MAX_LIVE`, кулдауны, cap глубины).

**Наблюдённые вживую сбои (2026-07-08):**
- `idle`+`chaining` дают низкоценный само-referential шум: байт-в-байт дубли (дедуп по id триггера, не по контенту — разные event-сигналы с одинаковым шаблонным текстом), рекурсия «развиваю дальше: «<родитель>»».
- Шумовые мысли («тишина уместнее слов») кормят wake-packet → **склоняют существо к `[SILENT]`** (самоусиливающаяся спираль тишины).
- Два независимых гейтинга → систему **не отладить** (напр. `skipped_busy` врёт про причину; см. lm-50b).

## 2. Решение (что делаем)

**Единая каузальная спина рождения durable-сущностей на быстром тике:**

```
сенсор(нейрон) → сигнал → generic Aggregation-раннер → cognition/act → intent-шина → SQLite
```

Ничто не рождает durable-сущности вне этой спины. `ThoughtGeneration` как параллельный движок — растворяется в спину.

**Ключевой процессный принцип: строим ИНКРЕМЕНТАЛЬНО.** Не закладываем весь каркас (generic-реестр политик, ack-хабитуация, TTL/prune, агрегация-счётчик, Вебер-эмиссия) наперёд под систему, у которой сегодня **один настоящий драйв (контакт) + тонкие event-мысли**. Механизм ядра добавляем, **когда новая фича его требует** — так форма обобщения будет известна, а не угадана (YAGNI), и направление легче корректировать. Целевая архитектура (§3) — компас; §5 режет её на «сейчас» и «по нужде».

## 3. Целевая архитектура (компас)

**Сенсоры (нейроны) — только измеряют, плоские нормализованные скаляры.** Нейрон не знает wake-порог агрегации («child doesn't know parent»); у него свой `emitThreshold` («есть ли давление, достойное сигнала»). Взвешивание факторов и «что родить» — **не** в нейроне.

**Aggregation = generic-раннер + реестр `BirthPolicy`.** Раннер диспатчит сигналы по типу. `BirthPolicy` — **чистая функция** `evaluate(signal, snapshot) -> candidate`, **без стейт-машины внутри**; знает свой `wakeThreshold`, **сама взвешивает** сырые факторы (`x⃗·w⃗`), свой `cooldown`, dedup-ключ, target-kind, планку «стоит рождать». Весь гейтинг — здесь, один раз.

**Cognition/Crystallization** — вниз по спине; births не создают (cognition запускает ход из активного желания; crystallization — Рубикон мысль→желание).

**Проекция промпта отделена от хранения.** Планка рождения = «стоит ли хранить». Отдельно — проекция: ранжирование по salience + **фильтр** (silence-biased/низко-релевантные вон) перед LLM, топ-N в «недавние мысли».

**Контент отделён от политики рождения** (раннер решает *рождать ли*; фабрика/шаблоны/позже-LLM строят *текст*).

**Конфликт-арбитр** (правило «обращение к человеку глушит одинокую руминацию») — **отдельный слой** между раннером и шиной (не внутри раннера), и **не сейчас** (в ближнем объёме контакт и event-мысли в тике взаимоисключающи).

### Механизмы из оригинала (портируем ПО НУЖДЕ)
- **Анти-флаппинг = Вебер-Фехнер:** нейрон эмитит по относительному изменению `log(u_t / u_last_emitted) > θ_weber`. **НО контакт — исключение** (см. инвариант «абсолютный контакт»).
- **Два порога:** нейрон `emitThreshold` (эмитит, пока давление держится) vs aggregation `wakeThreshold` (действие).
- **Рейт-лимит:** нейрон `refractory` + aggregation `cooldown`.
- **Ack-хабитуация:** сигнал `handled`/`deferred`(override на значимое изменение)/`suppressed` — через `AckIntent`, коммитится в SQLite, читается следующим тиком (см. инвариант «ack ≠ signals.consumed»).
- **Агрегация событий:** повторы склеиваются в одно событие с `aggregatedCount`+`firstOccurrence`.
- **TTL/prune** старых событий.
- **Давление = взвешенная сумма факторов**, но веса — в политике, не в нейроне.

## 4. Инварианты (обязательны, откуда бы ни начинали)

1. **Абсолютный контакт.** Контакт-wake остаётся **абсолютно-сертифицированным** (`sim/wake`): никакой relative/Вебер-фильтр не подавляет `u ≥ θ`. `ContactNeuron` эмитит raw `u` каждый тик. Вебер — только для несертифицированных/вторичных сенсоров. **Parity-тесты** контакт-политики против текущей `ContactAggregation`/`sim/wake` — обязательны перед заменой.
2. **Нейроны — плоские измерения;** взвешивание + политика рождения — в агрегации.
3. **Семантический ack — через SQLite intent/committer, НЕ `signals.consumed`.** `signals.consumed` остаётся транспортным read-дедупом. (Оговорка: шина метит consumed до коммита → краш между consume и commit теряет сигнал семантически; для replay критичных deferred — two-phase consume / импорт событий в SQLite до TTL.)
4. **Если вводим TTL/prune:** `TTL > max(cooldown, deferral) + skew`; control-сигналы (`taxonomy.CONTROL_KINDS`) не прунятся.
5. **Хранение мыслей ≠ проекция промпта.** Проекция фильтрует silence-biased/низко-релевантные. Инвариант: **скрытая (не спроецированная) мысль не влияет на рост драйва `u`** — `u` гасится только реальным обменом.
6. **Дедуп:** структурные — сигнал потреблён-один-раз (`origin_id`) + не рождать уже живую singleton-сущность. Для мыслей — **live content-fingerprint**: не рождать мысль, чей нормализованный отпечаток совпадает с **живой** мыслью (live-scope, НЕ глобальный хеш — иначе амнезия к легитимным повторам). Глобального контент-дедупа нет.
7. **Chaining — из быстрого тика вон, в Фазу 6** (офлайн-«сон»/консолидация: «читать граф мыслей → развивать/сливать»).
8. **`sim/` не трогаем** — генерализуем вызывающего вокруг сертифицированной математики, не математику.

## 5. Инкрементальный план (что «сейчас» vs «по нужде»)

### Сейчас (минимально-корректное ядро — чинит наблюдённые баги, сохраняет сертификат)
1. Растворить параллельное рождение мыслей в **чисто отрефакторенную агрегацию** (контакт + event-мысли в одной структуре) — **без** полного generic-реестра. Одна чистая точка рождения.
2. Контакт — абсолютный путь, **parity-тесты**.
3. Вырезать `chaining` + `idle` из тика → Фаза 6.
4. Event-мысли + **live-fingerprint** дедуп + де-биас шаблонов (silence-контент).
5. **Проекция промпта** (salience + фильтр) + инвариант «скрытая мысль не влияет на `u`».

### По нужде (извлекаем, когда фича/второй драйв реально требует)
- Generic-раннер + реестр `BirthPolicy` (извлечь обобщение при появлении **третьей** политики — форма будет известна).
- Ack-хабитуация (`deferred`/`suppressed`).
- TTL/prune + агрегация-счётчик событий.
- Вебер-эмиссия на вторичных сенсорах.
- Конфликт-арбитр (при втором вегетативном драйве).
- Богатая руминация / idle-мысли / chaining — **Фаза 6** (LLM-контент делает их осмысленными; нужен пороговый rumination-сигнал — как **давление**, не отдельный сенсор тишины, т.к. тишина — подмножество входа контакт-сенсора).

## 6. Отклонённые варианты
- **Полный порт control-theory аппарата сразу** — переусложнение под ~1 драйв (YAGNI); теряется гибкость направления.
- **Булеан-флаг `prompt-visible`** — костыль; заменён проекцией (ранжирование+фильтр) при разделённом хранении.
- **Глобальный контент-хеш дедуп** — амнезия к легитимным повторам; заменён live-fingerprint.
- **`RuminationNeuron` на «только тишине»** — подмножество входа контакт-сенсора; отдельный сенсор оправдан только под *другую* величину (давление открытых петель — Фаза 6).
- **Вебер-гейт на контакте** — убил бы сертифицированный абсолютный wake (Slow-Creep).

## 7. Открытые вопросы (на потом)
- Точная форма проекции промпта (предикат релевантности сверх salience).
- Two-phase consume vs SQLite-импорт событий — только если понадобится replay критичных deferred.
- Веса факторов давления (когда факторов станет >1).
