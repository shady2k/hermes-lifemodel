# Roadmap — hermes-lifemodel

**Статус:** draft v0.2 · **Дата:** 2026-07-03
**Уровень:** поставка (в каком порядке строим). «Что» — [BRD](business-requirements.md); «как» — [HLA](hla.md).

> Здесь мы **режем MVP** из целевого видения. Принцип (NG7): проектируем под продукт, но *поставляем узко и стадийно*; каждая фаза — отгружаемая и обкатывается автором; поздние фазы тянет **реальная нужда**.

## Ведущий принцип: сначала движок, потом личность

Сначала запускаем **голый движок end-to-end на одном нейроне** и проверяем, что слои вообще работают на Hermes. Всё, «без чего и так заработает» (genesis, полноценная душа, receptivity, открытые петли, энергия, стоимость, лестница разрешений), — **позже, по нужде**. Каждый шаг — маленький, независимо тестируемый (TDD).

Два сквозных правила (из ревью Codex):
- **Текст-only до Фазы 7:** когниция не делает `core.act`/код/действий в мире сверх своего состояния (полная лестница разрешений — Фаза 7).
- **Интерфейсы с первого дня:** уже в Фазе 1 модули за реальными интерфейсами (`StateStore` · `Neuron.tick(state)->signals` · `Aggregator(signals)->wake` · `WakePacket` §11 HLA · `ActGate` · `Delivery`); скрипт их *оркестрирует*, логику давления **не хардкодит** — иначе поздние фазы перепишут ранние.

## Фаза 0 — Каркас (минимум)

- **0.0** Тулчейн (HLA §13): uv · ruff · mypy --strict · pytest(+cov/mock/asyncio) · pre-commit · `pyproject.toml` · src-layout · единый гейт · **structlog**.
- **0.1** Скелет плагина (`register(ctx)`, грузится, per-profile).
- **0.2** Стор состояния: `StatePort` + JSON под профилем с **атомарной записью и версией схемы уже здесь** (даже в Фазе 1 два писателя: скрипт + пробуждённый ход). Жёсткую конкурентность/миграции/снэпшоты/merge (§9 HLA) — в Фазе 7.
- **0.3** Debug-дамп состояния (читает структурные события structlog).
- **0.4** Каркас гексагона: composition root (DI) + базовые классы `Neuron`/`Layer` (ABC) + `SignalBus` (durable лог: publish/consume) + порты, нужные Фазе 1 (`StatePort`/`DeliveryPort`/`ClockPort`). Прочие порты (`LlmPort`/`MemoryPort`) — по мере фаз (YAGNI). Чтобы Фаза 1 *наполняла* реализациями, а не переписывала.

## Фаза 1 — Голый движок · walking skeleton · **проверка «работает ли вообще»**

- **1.1 Heartbeat:** cron `every 1m` + `--script`; тик реально стреляет, скрипт читает/пишет стор.
- **1.2 Один нейрон (заглушка-таймер):** копит одно число давления каждый тик. Цель Фазы 1 — не поведение, а проверка движка; осмысленный нейрон «связь» — в 2.1.
- **1.3 Агрегация + порог:** скрипт решает «давление ≥ порога?» → отдаёт `wakeAgent`.
- **1.4 Пробуждение + доставка (с минимальной безопасностью):** на wake — минимальный ход когниции шлёт сообщение через gateway; давление слилось. Уже здесь: **только авторский/home-канал · ≤1 сообщение на цикл порога · кулдаун · без инструментов (текст-only) · без третьих лиц.**

*Проверяет:* слоёная петля **дышит end-to-end на Hermes**. Ноль личности. Самая дешёвая проверка тезиса.

**Критерии приёмки (Codex):** тик стрельнул ≥2 раза · давление персистит между тиками · порог отдал `wakeAgent` · пробуждённый ход получил wake-packet · одно сообщение доставлено в авторский канал · давление слилось/записано · **ниже порога — ноль LLM-вызовов**.

## Фаза 2 — Настоящий нейрон + две скорости

- **2.1** Нейрон «связь» (время без контакта) вместо заглушки.
- **2.2** fast → smart по уверенности (S1→S2).
- **2.3** Минимальный act-gate / `[SILENT]` (слать ли вообще).

## Фаза 3 — Тонкая душа

- **3.1** Soul-store: идентичность/характер прозой (пока умолчания, без genesis).
- **3.2** Soul-инъекция (`pre_llm_call` + уступающий `SOUL.md`) — чтобы сообщение звучало как *кто-то*.

## Фаза 4 — Genesis (знакомство)

- **4.1** Детект первого запуска + ритуал знакомства → соавторит soul-store. *Перенесено сюда: не основное — движок и душа уже работают на умолчаниях; знакомство их лишь персонализирует.*

## Фаза 5 — Желания и взросление

- **5.1** Открытые петли + receptivity (быстро улучшают поведение, дают субстрат сну — идут первыми).
- **5.2** SDT-драйвы целиком (веса темперамента).
- **5.3** Триггер/commitment-нейроны.
- **5.4** Opinions + predictions.

## Фаза 6 — Внутренняя жизнь и сон

- Мысли/руминация (thought-pressure, «Recent Thoughts»); dreaming (офлайн-консолидация).

## Фаза 7 — Доверие, безопасность, устойчивость

- **State API §9** (жёсткая конкурентность/лок), лестница разрешений (FR27), твёрдый пол (FR18), стойкость к инъекциям (NFR10), политика памяти/согласие (FR26), interrupt (FR29), деградация (FR28), энергия + потолок стоимости (FR20) как настоящие предохранители.

## Фаза 8 — Продуктовая поверхность

- Жизненный цикл души (FR11–16), мульти-user (FR9), упаковка (NFR7/8), genesis для незнакомца.

- Нейрон 1.2 = **заглушка-таймер** (движок, не поведение); «связь» → 2.1.
- Порядок 5↔6: желания раньше сна; петли/receptivity (5.1) — в самом начале.
- Минимальный State (атомик+версия) → Фаза 0; безопасность outbound → Фаза 1.4; текст-only до Фазы 7.
