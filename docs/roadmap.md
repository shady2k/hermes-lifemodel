# Roadmap — hermes-lifemodel

**Статус:** v0.6 · **Дата:** 2026-07-07
**Уровень:** поставка (в каком порядке строим). «Что» — [BRD](business-requirements.md); «как» — [HLA](hla.md).

> Здесь мы **режем MVP** из целевого видения. Принцип (NG7): проектируем под продукт, но *поставляем узко и стадийно*; каждая фаза — отгружаемая и обкатывается автором; поздние фазы тянет **реальная нужда**.

**Легенда статуса:** ✅ закрыто · ◐ частично / в работе · ○ впереди.
Каждая фаза = эпик в bd (label `roadmap`). Фазы связаны `blocks`-цепочкой: **следующая фаза не берётся в работу, пока предыдущая не закрыта**. Задачи на будущие фазы заводим при старте фазы, не заранее.

## Где мы сейчас: реальный путь

Роудмэп задумывался линейным («сначала движок, потом личность»). По факту мы прошли Фазы 0–1 и бо́льшую часть 2, а затем **ушли в глубокий вертикальный срез по одному драйву — «контакт»** и довели его до продакшена. Это и есть настоящая история проекта на сегодня:

1. **Каркас + голый движок** (Ф0–1) — гексагон, стор, heartbeat, нейрон-заглушка, порог, wake+доставка. → эпик «MVP — walking skeleton» ✅
2. **Проактивный egress** — существо тянется к человеку первым нативным assistant-турном (без cron-хрома и амнезии); доставка через reach-in. Shipped + live.
3. **Desire-модель контакта** — сертифицированная в `sim/` динамика желания (драйв `u`, жизненный цикл желания, safety-envelope), вживлённая в плагин («убийца барабана»). → суб-эпик «desire-модель» ✅
4. **Пересборка ядра** — монолит `decision.py` растворён в слоёный движок AUTONOMIC → AGGREGATION → COGNITION (Intents, единый владелец состояния, CoreLoop, энергия, backstop). → суб-эпик «пересборка ядра» ✅
5. **Существо как platform-adapter** — мозг хостится как supervised gateway-adapter; gateway владеет жизненным циклом и рестартит при падении; cron-heartbeat и «two-brain»-машинерия удалены. Merged.

**Следствие для роудмэпа:**
- Фаза 2 разрослась из «один нейрон + две скорости» в **первый полностью реализованный драйв**. «Две скорости» (2.2) воплотились не как S1→S2-эскалация, а как 0-LLM слои (быстро) → нативный ход Hermes по порогу (умно); act-gate (2.3) = собственный ход существа, `[SILENT]` = REJECT.
- Часть Фазы 7 **утянута вперёд**: энергия-физиология (`core/energy`) и fail-closed backstop-rate-limit (`core/backstop`) уже сделаны как часть desire-модели.
- Остаток Фазы 2 — **харднинг контакта**: живой импульс, наблюдаемость, инструменты дебага, хвосты ядра и egress (см. эпик «Фаза 2 — Контакт-драйв»).
- **Объектное BDI-ядро зафиксировано (HLA §4.1/D8, v0.8):** позвоночник **BDI** (Desire/Intention/Relationship/Thought), единая запись `memory_records` + типизированные модели-на-вид + машины состояний, мутация через шину интентов + атомарный коммиттер, Desire с **двумя источниками** (драйв + мысль), Thought — ограниченный генеративный поток. **Объектная модель — это НЕ Фаза 6, а базовая механика проактивности:** строится сейчас, эпик «Объектное ядро (BDI)» (`lm-27n`), поверх единой памяти. В Ф6 остаётся лишь надстройка (богатая руминация/сон/семантика/консолидация). Контракт отношений (личность с эмоциями, пользователь суверенен) — там же. Прозу-душу ведёт Hermes.

## Ведущий принцип: сначала движок, потом личность

Сначала запускаем **голый движок end-to-end на одном нейроне** и проверяем, что слои вообще работают на Hermes. Всё, «без чего и так заработает» (genesis, полноценная душа, receptivity, открытые петли, энергия, стоимость, лестница разрешений), — **позже, по нужде**. Каждый шаг — маленький, независимо тестируемый (TDD).

Два сквозных правила (из ревью Codex):
- **Текст-only до Фазы 7:** когниция не делает `core.act`/код/действий в мире сверх своего состояния (полная лестница разрешений — Фаза 7).
- **Интерфейсы с первого дня:** модули за реальными интерфейсами (`StatePort` · `Neuron` · `Aggregator` · `WakePacket` · act-gate · `Delivery`); скрипт их *оркестрирует*, логику давления **не хардкодит**.

## ◐ Фаза 2 — Контакт-драйв · *ядро готово, идёт харднинг*

Эпик «Фаза 2 — Контакт-драйв». Разрослась из «настоящий нейрон + две скорости» в первый полный драйв (см. «реальный путь» выше).

- **2.1** Нейрон «связь» (время без контакта) — ✅ `core/contact_neuron`.
- **2.2** Две скорости — ✅ переосмыслено: 0-LLM слои → нативный ход Hermes по порогу.
- **2.3** Act-gate / `[SILENT]` — ✅ собственный ход существа = act-gate.
- **Егress + доставка** — ✅ reach-in, нативный assistant-турн.
- **Desire-модель + слоёный движок** — ✅ (суб-эпики закрыты).
- **Хостинг как platform-adapter** — ✅.
- **Единая память (фундамент) — эпик `lm-fib.6`:** ✅ SQLite-стор + порты (`6.1`) и cutover `StatePort`, `state.json`/`JsonStateStore` retired (`6.2`) — отгружены (§4.1/D7 v0.7; codex ×6 + gemini; чинит дыру `state.json`→restart-loop). Осталось: recovery-once hardening (`lm-fib.6.5`, до Ф6). Объектная модель поверх памяти — **вынесена в отдельную фазу** (ниже).
- **Объектное ядро (BDI) — новая фаза, эпик `lm-27n`:** живая модель проактивного контакта (наука: BDI/Bratman, Gollwitzer/Рубикон, appraisal Lazarus/Scherer, SDT, responsiveness). Убивает флаг `desire_status`; объекты — типизированные строки `memory_records` с машинами состояний; мутация только через шину интентов + атомарный State+память коммиттер (снимок→интенты→коммит в конце тика). Живой минимум: **Desire + Intention + Relationship/Receptivity + Thought** (Commitment далее). Субстрат ядра → шина мутаций → Desire живьём (**RISKY**) → Intention → модель отношений → движок мыслей (6a-6d). **Наблюдаемость — сквозной инвариант ядра, а не хвост:** каждая мутация несёт и пропагирует каузальный трейс-контекст (W3C traceparent-совместимый: `trace_id`/`creation_span_id`/`parent_span_id`, stdlib-only, БЕЗ хард-зависимости на OTel — экспорт опциональным адаптером); `trace_id` сшивает durable-провенанс с эфемерными спанами. Поля — в `.1`, живая пропагация — в `.2`, поверхность (why-graph + опциональный OTel-экспорт) — `lm-27n.10`. Операционный дамп (статы/why-silent/reachability) остаётся `lm-9zj` (Ф2). Живой импульс (`lm-8o3`) — payoff этой фазы.
- **Единая спина рождения сущностей (single-spine) — HLA §8/D9, эпик `lm-md6`:** объектное ядро вскрыло архитектурный шов — рождение сущностей размазано на два независимых механизма (`ContactAggregation` из нейрона; и **отдельный** `ThoughtGeneration` со своими триггерами idle/chaining + своим гейтингом), что вживую дало спираль тишины, дубли мыслей и неотлаживаемость. Цель: одна каузальная спина `сенсор→сигнал→generic Aggregation-раннер→cognition→intent-шина→SQLite`, `ThoughtGeneration` растворяется в неё; механику событий/порогов портируем из оригинального `lifemodel`. **Строим ИНКРЕМЕНТАЛЬНО** (механизм ядра — когда фича его требует; generic-реестр/ack/TTL/агрегация — по нужде). Инварианты: контакт абсолютно-сертифицирован (`sim/wake`, parity), хранение мыслей ≠ проекция промпта, live-fingerprint дедуп, chaining/idle → Фаза 6. Полная запись: `docs/superpowers/specs/2026-07-08-core-single-spine-design.md`. Ближний срез чинит наблюдённые баги (спираль/дубли/двойной гейтинг), сохраняя сертификат.
- **Остаётся (харднинг):** **живой импульс** — `lm-8o3` (ситуативный бриф + скользящий *порог повода* при неизменных воротах уважения + деградация на свежей сессии + джиттер; читает операционное состояние **из памяти** — **ждёт фундамент**, blocked-by); **наблюдаемость** сверх HEALTH-строки — `lm-9zj` (дамп собранного брифа/`pressure-index`, «почему молчит»); хвосты ядра (§5.1/§8/§9.2, `duration_over_theta`, чистка ABC) и egress (smoke-проба, upstream, самоатрибуция импульса `lm-pbm`, lane-резолвинг).
- **Сделано:** инструменты дебага — force-wake/mutation (`lm-2vx`), `/lifemodel help` + список команд (`lm-67g`, `lm-fib.5`), читаемый формат дебага/help (`lm-fib.3/.4`); typing-фикс (`lm-fib.1`), force-wake/exchange (`lm-fib.2`).

## ○ Фаза 6 — Внутренняя жизнь и сон · *впереди*

Эпик «Фаза 6 — Внутренняя жизнь и сон».

- Мысли/руминация (thought-pressure, «Recent Thoughts»); dreaming (офлайн-консолидация).
- **Явно приземляется сюда (из single-spine, HLA/D9):** `chaining` («развить мысль» — чтение графа мыслей = рефлексия, не сенсор) и `idle`-руминация в текущем 0-LLM templated-виде дают шум/спираль тишины → **вырезаны из быстрого тика** и живут здесь (LLM-контент делает их осмысленными). Спонтанная idle-мысль — как **пороговое давление** через спину, не отдельный сенсор тишины.
- **Надстройка над объектным ядром** (само BDI-ядро — уже строится, эпик `lm-27n`, НЕ здесь): богатая руминация (генерация мыслей в простое, incubation), сон-консолидация (decay/forget/merge), семантический/векторный поиск + граф, `MemoryWorkspace` (read-your-writes для agentic-когниции). Плюс отложенное: Opinion/Prediction, семантика не-ответа, обучение предпочтениям, privacy-классификатор чувствительности, аудит «почему написал», забывание.
